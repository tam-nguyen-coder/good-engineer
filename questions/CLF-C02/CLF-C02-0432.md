# Câu hỏi #432

Which of the following is a cloud architectural design principle?

## Options

**A.** Scale up not out.
**B.** Loosely couple components.
**C.** Build monolithic systems.
**D.** Use commercial database software.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về design principle chuẩn của cloud architecture.
- **Existing Resources:** Không có cụ thể — bài toán khái niệm.
- **Current Issue/Goal:** Xác định principle đúng với cloud-native best practice.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Loosely couple components` | Decoupling — AWS Well-Architected principle, dùng SQS/SNS/EventBridge |
| `Scale up not out` | Trái với elasticity — scale out (nhiều instance nhỏ) thường tốt hơn scale up (1 instance lớn) |
| `Build monolithic systems` | Trái với design hiện đại, cloud khuyến khích microservices |
| `Use commercial database software` | Không phải design principle |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Khái niệm / Design principle
- **Constraints:** Single-answer

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- `Loose coupling` là nguyên tắc thiết kế cloud quan trọng: các component nên giao tiếp thông qua API/queue/event thay vì gọi trực tiếp (tight coupling), để một component fail không ảnh hưởng component khác.
- AWS hỗ trợ decoupling qua `SQS` (queue), `SNS` (pub/sub), `EventBridge` (event bus), `Step Functions` (orchestration).
- Đây là một trong những nguyên tắc cốt lõi của Reliability pillar.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- "Scale up not out" (tăng kích thước instance thay vì thêm instance) **trái** với cloud best practice.
- Đáp án này dẫn đến single point of failure và giới hạn scaling. Cloud thường ưu tiên scale out (horizontal scaling).

**❌ Đáp án C:**
- "Build monolithic systems" **trái** với modern cloud architecture (microservices, serverless, event-driven).
- Đáp án này là anti-pattern.

**❌ Đáp án D:**
- "Use commercial database software" không phải design principle, mà là lựa chọn công nghệ. Cloud có cả commercial (Oracle, SQL Server) và open-source (PostgreSQL, MySQL).
- Đáp án này không liên quan đến cloud architecture principle.

## 6. MẺO GHI NHỚ (Memory Hook)
🧠 *Cloud design principle: `loose coupling` + `elasticity` + `scalability` + `design for failure`. Tránh: `monolith`, `tight coupling`, `scale up not out`.*
