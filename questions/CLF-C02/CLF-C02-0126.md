# Câu hỏi #126

Which of the following AWS offerings is a MySQL-compatible relational database service that can scale capacity automatically based on demand?

## Options

**A.** Amazon Neptune.
**B.** Amazon Aurora.
**C.** Amazon RDS for SQL Server.
**D.** Amazon RDS for PostgreSQL.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _Which of the following AWS offerings is a MySQL-compatible relational database service that can scale capacity automatic..._
- **Existing Resources:** Đề nhắc tới: `RDS` — dịch vụ database quan hệ được quản lý.
- **Current Issue/Goal:** Hệ thống cần co giãn linh hoạt theo tải, không bị giới hạn bởi cấu hình cố định.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| ``RDS`` | dịch vụ database quan hệ được quản lý |
| ``Aurora`` | database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao |
| ``Neptune`` | graph database managed |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Scalability / Elasticity
- **Constraints:** Khả năng mở rộng theo tải

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
Đáp án `Aurora` đúng vì database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Aurora` đóng vai trò: RDS cao cấp với storage tự scale, 6 bản sao.

**❌ Đáp án A:**
- Tại sao sai: `Neptune` (graph database managed) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `Neptune`.

**❌ Đáp án C:**

- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `RDS`.

**❌ Đáp án D:**
- Tại sao sai: `RDS` (dịch vụ database quan hệ được quản lý) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `RDS`.
