# Câu hỏi #399

Which is a recommended pattern for designing a highly available architecture on AWS?

## Options

**A.** Ensure that components have low-latency network connectivity.
**B.** Run enough Amazon EC2 instances to operate at peak load.
**C.** Ensure that the application is designed to accommodate failure of any single component.
**D.** Use a monolithic application that handles all operations.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về khái niệm/dịch vụ AWS liên quan tới: _Which is a recommended pattern for designing a highly available architecture on ..._

- **Current Issue/Goal:** Lựa chọn giải pháp có hiệu năng cao nhất, độ trễ thấp nhất.

| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Highly available` | Yêu cầu hệ thống không bị gián đoạn |
| ``EC2`` | dịch vụ compute (máy chủ ảo) trên AWS Cloud |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance / Lowest latency
- **Constraints:** Hiệu năng và độ trễ thấp

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
Đáp án **C** mô tả đúng khái niệm/dịch vụ mà đề đang hỏi tới.
Đây là đáp án phù hợp nhất với best practice của AWS trong tình huống này.

- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy ứng dụng, web server, batch processing được ưu tiên hơn hiệu năng.
