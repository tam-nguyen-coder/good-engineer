# Câu hỏi #14

What are the benefits of having infrastructure hosted in AWS? (Choose TWO)

## Options

**A.** Increasing speed and agility.
**B.** There is no need to worry about security.
**C.** Gaining complete control over the physical infrastructure.
**D.** Operating applications on behalf of customers.
**E.** All of the physical security and most of the data/network security are taken care of for you.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về lợi ích khi chuyển infrastructure lên AWS.
- **Existing Resources:** Không có tài nguyên cụ thể.
- **Current Issue/Goal:** Xác định 2 lợi ích chính xác của việc dùng AWS Cloud.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `speed and agility` | Provisioning nhanh theo yêu cầu, iterate nhanh |
| `no need to worry about security` | Sai — khách hàng vẫn chịu trách nhiệm security in the cloud |
| `complete control over physical infrastructure` | Sai — AWS quản lý physical infra |
| `operating applications on behalf` | Sai — khách hàng tự vận hành app của mình |
| `physical security taken care` | Đúng — AWS chịu physical security |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Multi-answer (Choose TWO) — Lợi ích cloud
- **Constraints:** Phải là lợi ích chính đáng, không phải tuyên bố sai về security/control

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A và E**

**Giải thích:**
- **A — Increasing speed and agility:** Cloud cho phép provisioning tài nguyên chỉ trong vài phút thay vì vài tuần như on-premises, giúp tổ chức iterate nhanh, thử nghiệm ý tưởng mới và đưa sản phẩm ra thị trường nhanh hơn.
- **E — All of the physical security and most of the data/network security are taken care of for you:** AWS chịu trách nhiệm bảo vệ data center vật lý (giám sát, lực lượng bảo vệ, kiểm soát truy cập) và phần lớn bảo mật hạ tầng mạng/cơ sở vật chất. Khách hàng tập trung vào bảo mật application và data.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- Sai hoàn toàn. Khách hàng vẫn chịu trách nhiệm security in the cloud (data, IAM, OS patching, network config, ...). AWS không "lo liệu hết".

**❌ Đáp án C:**
- AWS là public cloud — khách hàng **không có** toàn quyền kiểm soát physical infrastructure. Đó là dịch vụ on-premises hoặc private cloud.

**❌ Đáp án D:**
- AWS cung cấp nền tảng, không vận hành ứng dụng thay khách hàng. Khách hàng tự build, deploy, vận hành app.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Cloud = `speed & agility` + AWS lo `physical security`. Cloud KHÔNG = hết lo security, hết kiểm soát physical, vận hành hộ."*
