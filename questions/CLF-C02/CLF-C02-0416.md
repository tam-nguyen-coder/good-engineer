# Câu hỏi #416

A company has deployed several relational databases on Amazon EC2 instances Every month the database software vendor releases new security patches that need to be applied to the databases. What is the MOST efficient way to apply the security patches?

## Options

**A.** Connect to each database instance on a monthly basis and download and apply the necessary security patches from the vendor.
**B.** Enable automate patching for the instances using the Amazon RDS console.
**C.** In AWS Config. configure a rule for the instances and the required patch level.
**D.** Use AWS Systems Manager to automate database patching according to a schedule.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Công ty có nhiều relational database chạy trên `Amazon EC2`. Hàng tháng vendor phát hành security patch.
- **Existing Resources:** Nhiều EC2 instance chạy database engine (MySQL, PostgreSQL…).
- **Current Issue/Goal:** Tìm cách áp dụng security patch **hiệu quả nhất** (ít thao tác thủ công nhất).

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `MOST efficient` | Tối ưu effort, tự động hóa cao |
| `EC2 instances` | Câu hỏi về EC2, không phải RDS managed |
| `AWS Systems Manager` | Dịch vụ quản lý fleet, có `Patch Manager` để patch EC2 theo lịch |
| `Amazon RDS console` | Không dùng được vì database này chạy trên EC2, không phải RDS managed |
| `AWS Config` | Theo dõi cấu hình, không tự patch |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Operational efficiency / Patch management
- **Constraints:** Database trên EC2 (không phải RDS); hàng tháng phát hành patch mới

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: D**

**Giải thích:**
- `AWS Systems Manager` (SSM) cung cấp `Patch Manager` cho phép tự động scan và patch OS/application trên hàng loạt EC2 instances theo `Maintenance Window` định sẵn.
- Bạn có thể định nghĩa `Patch Baseline` (vendor, severity, approve/reject list) và target qua `Resource Group` hoặc `Tag`.
- Vì databases chạy trên EC2 (không phải managed RDS), SSM Patch Manager là giải pháp chuẩn để tự động hóa patching thay vì SSH thủ công.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Connect thủ công từng instance tốn thời gian, dễ sai sót, không scale được khi fleet lớn.
- Đáp án này chỉ phù hợp với fleet rất nhỏ (1–2 instances).

**❌ Đáp án B:**
- "Enable automate patching using Amazon RDS console" **không áp dụng được** vì database chạy trên EC2, không phải `Amazon RDS` managed.
- Đáp án này chỉ đúng nếu câu hỏi nói về RDS.

**❌ Đáp án C:**
- `AWS Config` chỉ **đánh giá** cấu hình (ví dụ: rule kiểm tra xem instance có patch level phù hợp không) chứ **không tự động patch**.
- Đáp án này phù hợp cho compliance reporting, không phải patching.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *EC2 patching tự động = `AWS Systems Manager Patch Manager`. RDS patching tự động = `RDS Maintenance Window`. `Config` = đánh giá, không patch.*
