# Câu hỏi #2

Which of the following is an example of horizontal scaling in the AWS Cloud?

## Options

**A.** Replacing an existing EC2 instance with a larger, more powerful one.
**B.** Increasing the compute capacity of a single EC< instance to address the growing demands of an application.
**C.** Adding more RAM capacity to an EC2 instance.
**D.** Adding more EC2 instances of the same size to handle an increase in traffic.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về khái niệm scaling trong AWS Cloud, cụ thể là phân biệt giữa horizontal scaling và vertical scaling.
- **Existing Resources:** Đang có `EC2 instance` đang chạy một application.
- **Current Issue/Goal:** Xác định phương pháp scale đúng theo chiều ngang (horizontal scaling) — tức là tăng số lượng instance chứ không phải tăng cấu hình từng instance.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `horizontal scaling` | Thêm nhiều instance cùng loại (scale out) — mở rộng theo chiều ngang |
| `vertical scaling` | Tăng cấu hình (CPU, RAM) của một instance hiện có (scale up) — mở rộng theo chiều dọc |
| `more EC2 instances of the same size` | Dấu hiệu nhận biết horizontal scaling |
| `replace with larger, more powerful` | Dấu hiệu nhận biết vertical scaling |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Khái niệm cơ bản (Definition — scaling types)
- **Constraints:** Phải là horizontal scaling, không phải vertical scaling

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: D**

**Giải thích:**
- `Horizontal scaling` (scale out/in) nghĩa là thêm nhiều `EC2 instance` cùng cỡ để phân tán tải, ví dụ đặt sau `Elastic Load Balancer` kết hợp với `Auto Scaling Group`.
- Đáp án D thể hiện đúng: "thêm nhiều EC2 instance cùng cỡ để xử lý lưu lượng tăng".
- Đây là cách scaling phổ biến trong cloud vì dễ tự động hóa, không giới hạn bởi cấu hình phần cứng vật lý, và cho phép đạt high availability.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Đây là `vertical scaling` (scale up/down) — thay instance nhỏ bằng instance lớn hơn, không phải horizontal scaling.

**❌ Đáp án B:**
- Tăng compute capacity cho một instance cũng là `vertical scaling`. Câu này nói rõ "single EC2 instance", không thỏa mãn horizontal.

**❌ Đáp án C:**
- Thêm RAM cho một instance là thay đổi cấu hình phần cứng của một instance — đây cũng là `vertical scaling`.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Horizontal = thêm nhiều máy (số lượng). Vertical = thêm sức mạnh cho một máy (cấu hình). Thấy 'more instances' → horizontal."*
