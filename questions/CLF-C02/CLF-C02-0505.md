# Câu hỏi #505

For which auditing process does AWS have sole responsibility?

## Options

**A.** AWS IAM policies.
**B.** Physical security.
**C.** Amazon S3 bucket policies.
**D.** AWS CloudTrail Logs.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về khái niệm/dịch vụ AWS liên quan tới: _For which auditing process does AWS have sole responsibility?_
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn giải pháp bảo mật tốt nhất theo best practice của AWS.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Security` | Yêu cầu bảo mật cao |
| `Audit / Compliance` | Theo dõi, truy vết, kiểm toán |
| ``S3`` | dịch vụ lưu trữ object bền vững (11x 9s) |
| ``IAM`` | Identity and Access Management - quản lý quyền truy cập |
| ``CloudTrail`` | dịch vụ audit API call |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Most secure / Security best practice
- **Constraints:** Bảo mật dữ liệu và quyền truy cập

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
Đáp án **B** mô tả đúng khái niệm/dịch vụ mà đề đang hỏi tới.
Đây là đáp án phù hợp nhất với best practice của AWS trong tình huống này.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: `IAM` (Identity and Access Management - quản lý quyền truy cập) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là quản lý user, group, role, policy cho AWS resources này.

**❌ Đáp án C:**
- Tại sao sai: `S3` (dịch vụ lưu trữ object bền vững (11x 9s)) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là lưu trữ file, backup, static website, data lake, log archive này.

**❌ Đáp án D:**
- Tại sao sai: `CloudTrail` (dịch vụ audit API call) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là ghi lại mọi API call vào tài khoản AWS, hỗ trợ kiểm tra bảo mật này.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **B** = `S3`. Bảo mật = defense-in-depth: nhiều lớp, nguyên tắc least privilege.*
