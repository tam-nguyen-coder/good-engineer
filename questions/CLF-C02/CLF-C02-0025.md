# Câu hỏi #25

Which service provides DNS in the AWS cloud?

## Options

**A.** Route 53.
**B.** AWS Config.
**C.** Amazon CloudFront.
**D.** Amazon EMR.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về dịch vụ cung cấp chức năng `DNS` (Domain Name System) trong AWS Cloud.
- **Existing Resources:** Không có tài nguyên cụ thể.
- **Current Issue/Goal:** Xác định service DNS của AWS.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `DNS` | Hệ thống phân giải tên miền → IP address |
| `Route 53` | Dịch vụ DNS có khả năng scalability cao của AWS |
| `AWS Config` | Quản lý cấu hình tài nguyên, không phải DNS |
| `CloudFront` | CDN, phân phối nội dung, không phải DNS thuần |
| `Amazon EMR` | Big data processing với Hadoop/Spark, không phải DNS |

## 3. YÊU CẦU CỀ ĐỀ
- **Question type:** Khái niệm — DNS service
- **Constraints:** Phải là DNS service chính thống

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
- `Amazon Route 53` là dịch vụ DNS (Domain Name System) có khả năng mở rộng cao của AWS, tên lấy theo cổng 53 của DNS protocol.
- Hỗ trợ nhiều loại routing: `simple`, `weighted`, `latency-based`, `failover`, `geolocation`, `geoproximity`, `multi-value answer`.
- Ngoài ra còn cung cấp dịch vụ đăng ký tên miền (domain registrar) và health check.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- `AWS Config` là dịch vụ theo dõi và ghi lại thay đổi cấu hình của AWS resources, hoàn toàn không liên quan đến DNS.

**❌ Đáp án C:**
- `CloudFront` là CDN phân phối nội dung qua edge locations. Mặc dù có thể dùng alternate domain name (CNAME), bản chất CloudFront không phải DNS service.

**❌ Đáp án D:**
- `Amazon EMR` (Elastic MapReduce) là dịch vụ xử lý big data với Hadoop, Spark, Presto, ... — không liên quan DNS.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"DNS trong AWS = `Route 53` (port 53 = DNS). `CloudFront` = CDN, `Config` = config tracking, `EMR` = big data. Đừng nhầm!"*
