# Câu hỏi #38

Select TWO examples of the AWS shared controls.

## Options

**A.** Patch Management.
**B.** IAM Management.
**C.** VPC Management.
**D.** Configuration Management.
**E.** Data Center operations.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về `Shared Controls` trong `AWS Shared Responsibility Model`.
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi lý thuyết về trách nhiệm chung.
- **Current Issue/Goal:** Xác định 2 ví dụ đúng về shared controls.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Shared Controls` | Cả AWS lẫn khách hàng đều có trách nhiệm ở các layer khác nhau |
| `Patch Management` | AWS patch infra nền; khách hàng patch OS cho EC2; AWS patch cho managed services |
| `Configuration Management` | AWS quản lý config device; khách hàng quản lý config app/resource |
| `IAM Management` | Hoàn toàn thuộc khách hàng (không phải shared) |
| `VPC Management` | Hoàn toàn thuộc khách hàng |
| `Data Center operations` | Hoàn toàn thuộc AWS |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Multi-answer (Choose TWO) — Shared Responsibility
- **Constraints:** Phải là control mà cả AWS lẫn khách hàng đều chịu trách nhiệm

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A và D**

**Giải thích:**
- **A — Patch Management:** Shared control. AWS patch hạ tầng (hypervisor, host OS của managed service như RDS). Khách hàng patch `guest OS` của `EC2` và application. Cả hai cùng chịu ở các layer khác nhau.
- **D — Configuration Management:** Shared control. AWS duy trì cấu hình thiết bị hạ tầng của họ, khách hàng duy trì cấu hình resource AWS của mình (ví dụ: `Security Group`, `IAM policy`, `S3 bucket policy`).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- `IAM Management` hoàn toàn thuộc **khách hàng** — khách hàng tạo user, group, role, policy. AWS chỉ cung cấp cơ chế.

**❌ Đáp án C:**
- `VPC Management` hoàn toàn thuộc **khách hàng** — khách hàng tự thiết kế VPC, subnet, route table, NAT, ... AWS cung cấp khả năng nhưng khách hàng quyết định cấu hình.

**❌ Đáp án E:**
- `Data Center operations` (giám sát vật lý, an ninh, điện, làm mát, ...) hoàn toàn thuộc **AWS** — không phải shared control.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"`Shared` = `Patch` (mỗi bên patch layer mình) + `Configuration` (AWS device, khách hàng resource). `IAM`/`VPC` = khách hàng, `Data Center` = AWS."*
