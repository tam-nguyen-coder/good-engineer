# Câu hỏi #45

You have deployed your application on multiple Amazon EC2 instances. Your customers complain that sometimes they can't reach your application. Which AWS service allows you to monitor the performance of your EC2 instances to assist in troubleshooting these issues?

## Options

**A.** AWS Lambda.
**B.** AWS Config.
**C.** Amazon CloudWatch.
**D.** AWS CloudTrail.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Application deploy trên nhiều `EC2`. User phàn nàn thỉnh thoảng không truy cập được. Cần **monitor performance** của EC2 để troubleshoot.
- **Existing Resources:** Nhiều EC2 instance đang chạy app.
- **Current Issue/Goal:** Tìm công cụ AWS giám sát performance EC2 (CPU, network, status, log, ...) phục vụ troubleshooting.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `monitor the performance` | Cần metric, log, dashboard cho EC2 |
| `Amazon CloudWatch` | Dịch vụ monitoring & observability chính của AWS |
| `AWS Lambda` | Serverless compute, không phải monitoring |
| `AWS Config` | Theo dõi thay đổi cấu hình, không phải performance metric |
| `AWS CloudTrail` | Audit API call, không phải performance metric |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Chọn dịch vụ monitoring
- **Constraints:** Phải đo performance (metric, log) của EC2

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
- `Amazon CloudWatch` là dịch vụ monitoring & observability toàn diện của AWS, cho phép thu thập metric, log, event từ EC2 và nhiều resource khác.
- Mặc định, CloudWatch thu thập các metric cơ bản của EC2: `CPUUtilization`, `NetworkIn`, `NetworkOut`, `StatusCheckFailed`, `DiskRead/WriteBytes`, ... Có thể cài `CloudWatch Agent` để thu thập metric chi tiết hơn (memory, disk usage).
- Tích hợp với `CloudWatch Alarms` (cảnh báo), `CloudWatch Logs` (tập trung log), `CloudWatch Dashboards` (visualization), `CloudWatch Anomaly Detection` (ML-based).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `AWS Lambda` là dịch vụ serverless compute, cho phép chạy code theo event. Không phải công cụ monitoring.

**❌ Đáp án B:**
- `AWS Config` theo dõi thay đổi cấu hình resource và đánh giá compliance. Không thu thập metric performance theo thời gian thực.

**❌ Đáp án D:**
- `AWS CloudTrail` ghi lại API call (ai gọi, từ đâu, khi nào) cho mục đích audit/security, không phải performance metric.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Monitor performance = `CloudWatch` (metric + log + alarm). Audit = `CloudTrail`. Config tracking = `Config`. Đừng nhầm!"*
