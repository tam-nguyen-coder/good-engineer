# Câu hỏi #410

Which storage service can be used as a low-cost option for hosting static websites?

## Options

**A.** Amazon Glacier.
**B.** Amazon DynamoDB.
**C.** Amazon Elastic File System (Amazon EFS).
**D.** Amazon Simple Storage Service (Amazon S3).


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi xác định dịch vụ storage phù hợp để host một static website với chi phí thấp.
- **Existing Resources:** Cần một nơi lưu trữ HTML/CSS/JS/file tĩnh.
- **Current Issue/Goal:** Tìm dịch vụ lưu trữ rẻ, dễ dùng, hỗ trợ static website hosting.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `low-cost` | Rẻ, theo mức dùng (pay-as-you-go) |
| `static website` | File HTML/CSS/JS tĩnh, không cần server-side rendering |
| `Amazon S3` | Có tính năng `Static Website Hosting` tích hợp, rất rẻ |
| `Amazon Glacier` | Lưu trữ lạnh (archive), chi phí thấp nhưng có latency cao và phí retrieval |
| `Amazon DynamoDB` | NoSQL key-value, không host file tĩnh |
| `Amazon EFS` | File system NFS, chi phí cao hơn S3 cho use case web tĩnh |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Xác định dịch vụ storage cho static website
- **Constraints:** Phải low-cost, phù hợp static website

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: D**

**Giải thích:**
- `Amazon S3` hỗ trợ tính năng `Static Website Hosting` cho phép serve file HTML, CSS, JS, hình ảnh trực tiếp từ bucket thông qua HTTP/HTTPS endpoint (ví dụ: `http://bucket-name.s3-website-region.amazonaws.com`).
- Chi phí cực thấp: chỉ trả tiền cho storage (vài cent/GB/tháng) và request/data transfer. Không cần chạy server, không cần patching.
- Có thể kết hợp với `CloudFront` để CDN, với `Route 53` để custom domain, với `ACM` để có HTTPS.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `Amazon Glacier` là storage lạnh cho archive (chi phí rất thấp), nhưng có retrieval latency từ phút đến giờ và phí retrieval. Không phù hợp để serve request web trực tiếp.
- Đáp án này dùng cho backup dài hạn, compliance archive.

**❌ Đáp án B:**
- `Amazon DynamoDB` là NoSQL database, không thể host file tĩnh (HTML/CSS).
- Đáp án này dùng cho data store với millisecond latency.

**❌ Đáp án C:**
- `Amazon EFS` (Elastic File System) là NFS file system cho Linux, cần EC2 để gắn và serve. Chi phí cao hơn S3 và tốn thêm compute.
- Đáp án này phù hợp cho shared filesystem giữa nhiều EC2 (ví dụ: WordPress cluster, content management).

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Static website rẻ nhất = `S3 Static Website Hosting` + có thể kết hợp `CloudFront`. `Glacier` = archive, không host web. `EFS` cần EC2.*
