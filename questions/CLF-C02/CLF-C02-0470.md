# Câu hỏi #470

Which service is best for storing common database query results, which helps to alleviate database access load?

## Options

**A.** Amazon Machine Learning.
**B.** Amazon SQS.
**C.** Amazon ElastiCache.
**D.** Amazon EC2 Instance Store.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _Which service is best for storing common database query results, which helps to alleviate database access load?_
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn giải pháp bảo mật tốt nhất theo best practice của AWS.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Machine learning` | Ứng dụng ML/AI |
| ``EC2`` | dịch vụ compute (máy chủ ảo) trên AWS Cloud |
| ``SQS`` | Simple Queue Service - message queue |
| ``ElastiCache`` | in-memory cache (Redis, Memcached) |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Most secure / Security best practice
- **Constraints:** Bảo mật dữ liệu và quyền truy cập

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
Đáp án `ElastiCache` đúng vì in-memory cache (Redis, Memcached), phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `ElastiCache` đóng vai trò: tăng tốc ứng dụng bằng cache, giảm tải database.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: `SageMaker` (ML platform managed) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là xây dựng, train, deploy machine learning model này.

**❌ Đáp án B:**
- Tại sao sai: `SQS` (Simple Queue Service - message queue) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là decouple các component, buffer message, xử lý bất đồng bộ này.

**❌ Đáp án D:**
- Tại sao sai: `EC2` (dịch vụ compute (máy chủ ảo) trên AWS Cloud) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu bảo mật chính là chạy ứng dụng, web server, batch processing này.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **C** = `EC2`. Bảo mật = defense-in-depth: nhiều lớp, nguyên tắc least privilege.*
