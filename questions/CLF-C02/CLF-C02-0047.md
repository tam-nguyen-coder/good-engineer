# Câu hỏi #47

Which of the following is not a benefit of Amazon S3? (Choose TWO)

## Options

**A.** Amazon S3 provides unlimited storage for any type of data.
**B.** Amazon S3 can run any type of application or backend system.
**C.** Amazon S3 stores any number of objects, but with object size limits.
**D.** Amazon S3 can be scaled manually to store and retrieve any amount of data from anywhere.
**E.** Amazon S3 provides 99.999999999% (11 9's) of data durability.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về những mô tả **KHÔNG phải** lợi ích của `Amazon S3`.
- **Existing Resources:** Không có tài nguyên cụ thể.
- **Current Issue/Goal:** Xác định 2 mô tả sai về S3.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Amazon S3` | Object storage, không phải compute, không chạy app |
| `unlimited storage` | Đúng — S3 có dung lượng lưu trữ không giới hạn |
| `run any application` | Sai — S3 là storage, không phải compute |
| `object size limits` | Đúng — object max 5 TB; số object không giới hạn |
| `scaled manually` | Sai — S3 **tự động scale** |
| `99.999999999% (11 9's) durability` | Đúng — S3 thiết kế cho durability cực cao |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Multi-answer (Choose TWO) — Negative identification
- **Constraints:** Phải là mô tả KHÔNG đúng về S3

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B và D**

**Giải thích:**
- **B — Amazon S3 can run any application or backend system:** Sai. S3 là **object storage** dùng để lưu trữ file, không phải compute service. S3 không thể "run" application. Để chạy code, cần `EC2`, `Lambda`, `ECS`, ...
- **D — Amazon S3 can be scaled manually:** Sai. S3 **tự động scale** để đáp ứng mọi lượng dữ liệu và traffic mà không cần khách hàng thao tác scale. Đây là một trong những lợi thế lớn nhất của S3.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `Unlimited storage` là **đúng** — S3 cho phép lưu trữ khối lượng dữ liệu không giới hạn về tổng dung lượng và số object.

**❌ Đáp án C:**
- `Object size limits` là **đúng** — mỗi object S3 có thể lớn từ 0 byte đến tối đa 5 TB. Số lượng object trong bucket là không giới hạn.

**❌ Đáp án E:**
- `99.999999999% (11 9's) durability` là **đúng** — S3 được thiết kế cho durability cực cao nhờ replicate dữ liệu across nhiều `AZ` và nhiều thiết bị. Con số này đại diện cho xác suất **không mất** object trong một năm.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"S3 = object storage (KHÔNG chạy app) + auto-scale (KHÔNG manual). Đúng: unlimited, 5TB max object, 11 9's durability."*
