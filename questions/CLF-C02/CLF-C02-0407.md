# Câu hỏi #407

Which AWS service of feature can be used to monitor CPU usage?

## Options

**A.** AWS CloudTrail.
**B.** VPC Flow Logs.
**C.** Amazon CloudWatch.
**D.** AWSConfig.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về dịch vụ giám sát (monitoring) CPU usage.
- **Existing Resources:** Không rõ — câu hỏi khái niệm.
- **Current Issue/Goal:** Xác định dịch vụ phù hợp để thu thập và theo dõi số liệu CPU.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `monitor CPU usage` | Metrics thuộc về CloudWatch |
| `Amazon CloudWatch` | Dịch vụ giám sát metrics, logs, alarms cho AWS resources |
| `AWS CloudTrail` | Ghi lại API calls (audit log) — không phải metrics |
| `VPC Flow Logs` | Ghi IP traffic của network interface, không phải CPU |
| `AWS Config` | Theo dõi cấu hình và sự thay đổi tài nguyên, không phải CPU metrics |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Xác định dịch vụ monitoring
- **Constraints:** Cần theo dõi metric CPU (định lượng)

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
- `Amazon CloudWatch` là dịch vụ giám sát tập trung của AWS, cung cấp metrics (CPU, network, disk…), logs, events và alarms.
- Mặc định, CloudWatch thu thập `CPUUtilization` của EC2 mỗi 5 phút (hoặc 1 phút với detailed monitoring). Bạn cũng có thể tạo custom metrics.
- Có thể thiết lập `CloudWatch Alarm` để cảnh báo khi CPU vượt ngưỡng và kích hoạt `Auto Scaling` để tự động thêm/bớt instance.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `AWS CloudTrail` ghi lại ai đã gọi API nào, khi nào, từ đâu — phục vụ audit và security investigation, không đo CPU.
- Đáp án này trả lời cho câu hỏi "ai đã làm gì với account này".

**❌ Đáp án B:**
- `VPC Flow Logs` ghi lại IP traffic đi qua network interface trong VPC (source/dest IP, port, accept/reject), không liên quan đến CPU.
- Đáp án này dùng cho network troubleshooting và security analysis.

**❌ Đáp án D:**
- `AWS Config` theo dõi sự thay đổi cấu hình tài nguyên (ví dụ: security group có rule mở SSH 0.0.0.0/0), không đo CPU runtime.
- Đáp án này phù hợp cho compliance và configuration drift detection.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *CPU/metrics = `CloudWatch`. API calls = `CloudTrail`. Network traffic = `VPC Flow Logs`. Config change = `AWS Config`.*
