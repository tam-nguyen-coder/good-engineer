# Câu hỏi #403

Which of the following AWS Cloud services can be used to run a customer-managed relational database?

## Options

**A.** Amazon EC2.
**B.** Amazon Route 53.
**C.** Amazon ElastiCache.
**D.** Amazon DynamoDB.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi xác định dịch vụ AWS cho phép khách hàng tự quản lý một relational database.
- **Existing Resources:** Không có — bài toán khái niệm.
- **Current Issue/Goal:** Tìm dịch vụ cung cấp quyền kiểm soát hoàn toàn (customer-managed) cho database quan hệ.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `customer-managed` | Khách hàng tự chịu trách nhiệm patching, backup, OS, vận hành |
| `relational database` | DB quan hệ dạng SQL: MySQL, PostgreSQL, Oracle, SQL Server… |
| `Amazon EC2` | Cung cấp máy ảo — có thể cài đặt bất kỳ DB engine nào |
| `Amazon DynamoDB` | NoSQL managed, không phải relational |
| `Amazon ElastiCache` | In-memory cache (Redis/Memcached), không phải relational DB |
| `Amazon Route 53` | DNS service, không liên quan đến database |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Xác định dịch vụ / Phân biệt managed vs unmanaged
- **Constraints:** Phải là "customer-managed" (không phải managed service) và "relational"

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
- `Amazon EC2` cung cấp máy chủ ảo, cho phép khách hàng cài đặt và quản lý hoàn toàn bất kỳ relational database engine nào (MySQL, PostgreSQL, Oracle, SQL Server…).
- Khách hàng chịu trách nhiệm toàn bộ: cài OS, patching, backup, replication, high availability. Đây chính là "customer-managed relational database".
- Đây là lựa chọn khi cần toàn quyền kiểm soát DB hoặc cần một tính năng không có trong `Amazon RDS`.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- `Amazon Route 53` là dịch vụ DNS, không liên quan đến việc chạy database.
- Đáp án này dùng để phân giải tên miền thành IP.

**❌ Đáp án C:**
- `Amazon ElastiCache` là in-memory cache (Redis/Memcached), dùng để tăng tốc truy vấn chứ không phải relational database.
- Đáp án này hỗ trợ caching, session store — không phải primary data store quan hệ.

**❌ Đáp án D:**
- `Amazon DynamoDB` là NoSQL key-value/document store, không phải relational database.
- Đáp án này phù hợp cho workload cần single-digit millisecond latency ở quy mô cực lớn với schema linh hoạt.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Customer-managed relational DB = `EC2` (tự cài MySQL/Postgres). Cần managed relational DB = `RDS`. Cần NoSQL = `DynamoDB`.*
