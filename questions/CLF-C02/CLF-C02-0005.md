# Câu hỏi #5

Which statement is true regarding the AWS Shared Responsibility Model?

## Options

**A.** Responsibilities vary depending on the services used.
**B.** Security of the IaaS services is the responsibility of AWS.
**C.** Patching the guest OS is always the responsibility of AWS.
**D.** Security of the managed services is the responsibility of the customer.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về `AWS Shared Responsibility Model` — mô hình chia sẻ trách nhiệm bảo mật giữa AWS và khách hàng.
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi lý thuyết.
- **Current Issue/Goal:** Xác định phát biểu đúng về cách phân chia trách nhiệm bảo mật.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Shared Responsibility Model` | AWS chịu trách nhiệm "security OF the cloud"; khách hàng chịu "security IN the cloud" |
| `vary depending on services` | Mức độ trách nhiệm khách hàng thay đổi theo từng dịch vụ (IaaS/PaaS/SaaS) |
| `IaaS` | Khách hàng chịu nhiều trách nhiệm hơn (OS, network, app) |
| `managed services` | AWS chịu nhiều hơn (RDS, Lambda, ...), khách hàng chịu data & access |
| `patching guest OS` | Khách hàng tự chịu với EC2, AWS chịu với RDS/Lambda |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Khái niệm — Shared Responsibility Model
- **Constraints:** Câu nào phản ánh đúng bản chất "tráchch nhiệm thay đổi theo dịch vụ"

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
- Trong `Shared Responsibility Model`, phần trách nhiệm của khách hàng thay đổi tùy theo dịch vụ: với `EC2` (IaaS) khách hàng chịu OS patching, network, firewall; với `RDS` (managed) AWS chịu OS patching; với `Lambda` AWS chịu gần như toàn bộ runtime.
- Do đó phát biểu "Responsibilities vary depending on the services used" là phát biểu đúng và tổng quát nhất.
- AWS luôn chịu trách nhiệm "Security OF the Cloud" (cơ sở hạ tầng vật lý, phần cứng, global infrastructure).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- Với `IaaS` (như EC2), khách hàng mới là người chịu trách nhiệm về security trong hệ điều hành, network ACL, ... AWS chỉ chịu phần dưới hypervisor.

**❌ Đáp án C:**
- Patching `guest OS` của `EC2` là trách nhiệm của khách hàng (vì EC2 là IaaS). Chỉ với managed service (RDS, Aurora, ...) thì AWS mới chịu.

**❌ Đáp án D:**
- Với `managed services`, AWS chịu phần lớn vận hành và bảo mật hạ tầng, khách hàng chỉ chịu trách nhiệm về data, access control và cách dùng service.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"IaaS (EC2) = bạn lo OS + patch; PaaS (RDS) = AWS lo OS; SaaS = AWS lo gần hết. Câu đúng luôn là: 'trách nhiệm thay đổi theo dịch vụ'."*
