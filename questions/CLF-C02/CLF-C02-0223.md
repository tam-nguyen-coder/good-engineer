# Câu hỏi #223

Which AWS Service helps enterprises extend their on-premises storage to AWS in a cost-effective manner?

## Options

**A.** AWS Data Pipeline.
**B.** AWS Storage Gateway.
**C.** Amazon Aurora.
**D.** Amazon EFS.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _Which AWS Service helps enterprises extend their on-premises storage to AWS in a cost-effective manner?_
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn giải pháp giúp tối ưu/tiết kiệm chi phí nhất mà vẫn đáp ứng yêu cầu nghiệp vụ.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Hybrid / On-premises` | Kết nối môi trường on-premises với AWS |
| ``EFS`` | Elastic File System - file system NFS managed |
| ``Storage Gateway`` | cầu nối lưu trữ hybrid |
| ``Aurora`` | database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Most cost-effective / Cost optimization
- **Constraints:** Ngân sách/chi phí là yếu tố quan trọng

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
Đáp án `Storage Gateway` đúng vì cầu nối lưu trữ hybrid, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Storage Gateway` đóng vai trò: kết nối on-premises với S3, Glacier qua cache local.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: Đáp án này không trực tiếp giải quyết yêu cầu cụ thể của đề, hoặc dịch vụ/tính năng này có giới hạn so với đáp án đúng.
- Khi nào đúng: Đáp án này sẽ đúng trong bối cảnh khác, nơi ưu tiên khác với yêu cầu của câu hỏi này.

**❌ Đáp án C:**
- Tại sao sai: `Aurora` (database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi ưu tiên RDS cao cấp với storage tự scale, 6 bản sao mà không yêu cầu tối ưu chi phí.

**❌ Đáp án D:**
- Tại sao sai: `EFS` (Elastic File System - file system NFS managed) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi ưu tiên chia sẻ file giữa nhiều EC2, Linux, scale tự động mà không yêu cầu tối ưu chi phí.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **B** = `EFS`. Tiết kiệm = pay-as-you-go, đừng mua hạ tầng thừa.*
