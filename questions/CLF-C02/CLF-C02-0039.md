# Câu hỏi #39

In order to implement best practices when dealing with a 'Single Point of Failure,' you should attempt to build as much automation as possible in both detecting and reacting to failure. Which of the following AWS services would help? (Choose TWO)

## Options

**A.** ELB.
**B.** Auto Scaling.
**C.** Amazon Athen.
**D.** ECR.
**E.** Amazon EC2.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đang đối phó với `Single Point of Failure (SPOF)`. Cần **tự động phát hiện** (detect) và **tự động phản ứng** (react) khi có failure.
- **Existing Resources:** Hệ thống có thể đang có SPOF.
- **Current Issue/Goal:** Loại bỏ SPOF bằng automation detect + react to failure.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Single Point of Failure` | Thành phần duy nhất — lỗi = toàn hệ thống chết |
| `detect and react to failure` | Health check + tự thay thế instance hỏng |
| `ELB (Elastic Load Balancer)` | Health check EC2, loại bỏ instance unhealthy khỏi rotation |
| `Auto Scaling` | Tự thay thế instance hỏng bằng instance mới |
| `Amazon Athena` | Query SQL trên S3, không liên quan HA |
| `ECR` | Container registry, không phải HA automation |
| `EC2` | Resource thụ động, không tự phản ứng failure |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Multi-answer (Choose TWO) — HA automation
- **Constraints:** Phải giúp detect + react tự động với failure

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A và B**

**Giải thích:**
- **A — ELB:** Thực hiện health check định kỳ trên các EC2 instance trong target group. Khi phát hiện instance unhealthy, ELB tự động loại bỏ instance đó khỏi rotation — chỉ gửi traffic đến instance khỏe mạnh.
- **B — Auto Scaling:** Khi instance unhealthy (do ELB phát hiện hoặc health check riêng), Auto Scaling tự động **terminate instance hỏng** và **launch instance mới** để thay thế. Kết hợp với `CloudWatch alarms` để scale theo metric tùy biến.
- Kết hợp ELB + Auto Scaling + Health Check tạo vòng tự động phát hiện & phản ứng failure, loại bỏ SPOF.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án C:**
- `Amazon Athena` là serverless query service cho phép chạy SQL trên S3, không liên quan đến HA hay automation failure.

**❌ Đáp án D:**
- `Amazon ECR` (Elastic Container Registry) lưu trữ Docker image, không có cơ chế detect/react failure.

**❌ Đáp án E:**
- `Amazon EC2` là compute service cơ bản, bản thân nó không tự phát hiện hay tự thay thế khi chính nó bị lỗi. Cần ELB + Auto Scaling để trở nên HA.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Chống SPOF: `ELB` health check (detect) + `Auto Scaling` thay thế (react). `EC2`/`Athena`/`ECR` không có cơ chế tự phục hồi."*
