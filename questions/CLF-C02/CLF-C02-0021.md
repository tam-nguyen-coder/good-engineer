# Câu hỏi #21

What does Amazon CloudFront use to distribute content to global users with low latency?

## Options

**A.** AWS Global Accelerator.
**B.** AWS Regions.
**C.** AWS Edge Locations.
**D.** AWS Availability Zones.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về cơ sở hạ tầng mà `Amazon CloudFront` dùng để phân phối nội dung đến user toàn cầu với độ trễ thấp.
- **Existing Resources:** `CloudFront` CDN đang phân phối content.
- **Current Issue/Goal:** Xác định thành phần hạ tầng chịu trách nhiệm về khả năng low-latency của CloudFront.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `distribute content globally` | CDN phân phối nội dung tới user toàn cầu |
| `low latency` | Yêu cầu đặt nội dung gần user nhất có thể |
| `Edge Locations` | Hàng trăm điểm hiện diện của AWS, nơi CloudFront cache nội dung |
| `AWS Global Accelerator` | Dùng `Anycast IP` cho application, không phải CDN cache |
| `Regions / AZs` | Cấp Region/data center, xa hơn edge location nhiều |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Khái niệm CloudFront infrastructure
- **Constraints:** Phải là thành phần hạ tầng gần user nhất

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
- `AWS Edge Locations` là các điểm hiện diện (PoP) rải rác khắp thế giới, là nơi `CloudFront` cache nội dung tĩnh và streaming (hình ảnh, video, JS, CSS, ...).
- Khi user truy cập, request được định tuyến tới edge location gần nhất → round-trip time thấp → trải nghiệm tốt.
- Hiện tại AWS có hàng trăm edge locations, nhiều hơn số Regions rất nhiều, tối ưu cho CDN.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `AWS Global Accelerator` cũng dùng edge locations nhưng mục đích là định tuyến TCP/UDP tới application endpoint (EC2, ALB, EIP) qua AWS global network, không phải CDN cache nội dung.

**❌ Đáp án B:**
- `Regions` là vùng địa lý lớn, có nhiều data center. Khoảng cách từ user tới Region lớn hơn nhiều so với edge location, latency cao hơn.

**❌ Đáp án D:**
- `Availability Zones` là data center trong Region, càng xa user hơn và phục vụ mục đích HA/fault tolerance, không phải CDN edge.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"`CloudFront` = `Edge Locations` cache. `Regions`/`AZs` = data center. `Global Accelerator` = routing, không phải CDN."*
