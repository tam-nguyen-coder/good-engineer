# Câu hỏi #445

Which of the following is a fast and reliable NoSQL database service?

## Options

**A.** Amazon Redshift.
**B.** Amazon RDS.
**C.** Amazon DynamoDB.
**D.** Amazon S3.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về khái niệm/dịch vụ AWS liên quan tới: _Which of the following is a fast and reliable NoSQL database service?_
- **Existing Resources:** Đề nhắc tới: `DynamoDB` — dịch vụ NoSQL key-value được quản lý hoàn toàn.
- **Current Issue/Goal:** Lựa chọn giải pháp có hiệu năng cao nhất, độ trễ thấp nhất.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Performance` | Yêu cầu hiệu năng, độ trễ thấp |
| ``S3`` | dịch vụ lưu trữ object bền vững (11x 9s) |
| ``RDS`` | dịch vụ database quan hệ được quản lý |
| ``DynamoDB`` | dịch vụ NoSQL key-value được quản lý hoàn toàn |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance / Lowest latency
- **Constraints:** Hiệu năng và độ trễ thấp

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
Đáp án `DynamoDB` đúng vì dịch vụ NoSQL key-value được quản lý hoàn toàn, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `DynamoDB` đóng vai trò: lưu trữ dữ liệu phi quan hệ với độ trễ thấp ở mọi quy mô.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: `Redshift` (data warehouse columnar) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu phân tích dữ liệu lớn, OLAP, BI được ưu tiên hơn hiệu năng.

**❌ Đáp án B:**
- Tại sao sai: `RDS` (dịch vụ database quan hệ được quản lý) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy SQL database (MySQL, PostgreSQL, Oracle, SQL Server, MariaDB, Aurora) được ưu tiên hơn hiệu năng.

**❌ Đáp án D:**
- Tại sao sai: `S3` (dịch vụ lưu trữ object bền vững (11x 9s)) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu lưu trữ file, backup, static website, data lake, log archive được ưu tiên hơn hiệu năng.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **C** = `S3`. Hiệu năng = edge location + cache + chọn instance/region gần user.*
