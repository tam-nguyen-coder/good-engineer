# Câu hỏi #279

A company experiences fluctuations in traffic patterns to their e-commerce website when running flash sales. What service can help the company dynamically match the required compute capacity to handle spikes in traffic during flash sales?

## Options

**A.** AWS Auto Scaling.
**B.** Amazon Elastic Compute Cloud.
**C.** Amazon Elastic File System.
**D.** Amazon ElastiCache.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Tình huống được mô tả: _A company experiences fluctuations in traffic patterns to their e-commerce website when running flash sales. What servic..._
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn giải pháp có hiệu năng cao nhất, độ trễ thấp nhất.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| ``EC2`` | dịch vụ compute (máy chủ ảo) trên AWS Cloud |
| ``EFS`` | Elastic File System - file system NFS managed |
| ``Auto Scaling`` | tự động scale số lượng EC2 |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance / Lowest latency
- **Constraints:** Hiệu năng và độ trễ thấp

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
Đáp án `Auto Scaling` đúng vì tự động scale số lượng EC2, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Auto Scaling` đóng vai trò: thêm/giảm instance theo demand, đảm bảo availability.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- Tại sao sai: `EC2` (dịch vụ compute (máy chủ ảo) trên AWS Cloud) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy ứng dụng, web server, batch processing được ưu tiên hơn hiệu năng.

**❌ Đáp án C:**
- Tại sao sai: `EFS` (Elastic File System - file system NFS managed) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chia sẻ file giữa nhiều EC2, Linux, scale tự động được ưu tiên hơn hiệu năng.

**❌ Đáp án D:**
- Tại sao sai: `ElastiCache` (in-memory cache (Redis, Memcached)) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu tăng tốc ứng dụng bằng cache, giảm tải database được ưu tiên hơn hiệu năng.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **A** = `EC2`. Hiệu năng = edge location + cache + chọn instance/region gần user.*
