# Câu hỏi #433

Which service enables risk auditing by continuously monitoring and logging account activity, including user actions in the AWS Management Console and AWS SDKs?

## Options

**A.** Amazon CloudWatch.
**B.** AWS CloudTrail.
**C.** AWS Config.
**D.** AWS Health.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về dịch vụ audit/ghi log hoạt động account.
- **Existing Resources:** Một AWS account với user actions.
- **Current Issue/Goal:** Cần audit trail cho risk auditing, security investigation.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `risk auditing` | Audit bảo mật, compliance |
| `continuously monitoring and logging account activity` | API audit log |
| `AWS CloudTrail` | Ghi lại tất cả API calls (Console, SDK, CLI) |
| `Amazon CloudWatch` | Metrics & logs (application/OS), không phải API audit |
| `AWS Config` | Theo dõi config changes, không phải user actions |
| `AWS Health` | Trạng thái dịch vụ AWS, alerts về sự cố |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Xác định dịch vụ audit
- **Constraints:** Phải ghi log user actions (Console + SDK)

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- `AWS CloudTrail` ghi lại toàn bộ API calls từ AWS Management Console, AWS SDK, AWS CLI, và AWS services do AWS tự thực hiện.
- Mỗi event bao gồm: actor (user/role), event time, source IP, request parameters, response elements, resources affected.
- Có thể lưu log lâu dài trong S3, tích hợp CloudWatch Logs/EventBridge, query bằng Athena/CloudTrail Lake.
- Đây là công cụ chuẩn cho security investigation, compliance audit, troubleshooting.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `Amazon CloudWatch` thu thập metrics, logs (application/OS), events, alarms — không phải audit trail của API calls.
- Đáp án này phù hợp cho monitoring runtime.

**❌ Đáp án C:**
- `AWS Config` theo dõi configuration changes của resources (vd: security group rules thay đổi thế nào) và compliance evaluation. Không track user actions ở mức API.
- Đáp án này phù hợp cho configuration compliance.

**❌ Đáp án D:**
- `AWS Health` cung cấp dashboard và alerts về tình trạng AWS services, lịch sử sự cố planned/unplanned. Không ghi log API calls.
- Đáp án này phù hợp cho service health awareness.

## 6. MẺO GHI NHỚ (Memory Hook)
🧠 *Audit API calls (ai gọi gì, khi nào) = `CloudTrail`. Config changes (resource thay đổi thế nào) = `Config`. Service health (AWS gặp sự cố gì) = `Health`.*
