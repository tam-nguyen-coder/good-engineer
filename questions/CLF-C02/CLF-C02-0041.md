# Câu hỏi #41

A developer is planning to build a two-tier web application that has a MySQL database layer. Which of the following AWS database services would provide automated backups for the application?

## Options

**A.** A MySQL database installed on an EC2 instance.
**B.** Amazon Aurora.
**C.** Amazon DynamoDB.
**D.** Amazon Neptune.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Một developer đang xây dựng ứng dụng web 2-tier với **MySQL database**. Cần chọn AWS database service có **automated backups**.
- **Existing Resources:** Web application cần MySQL backend.
- **Current Issue/Goal:** Database service có sẵn backup tự động, giảm công việc quản trị.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `MySQL database` | RDBMS tương thích MySQL cần thiết |
| `automated backups` | Backup tự động hàng ngày, giữ retention window |
| `Amazon Aurora` | MySQL/PostgreSQL-compatible, fully managed, có automated backup |
| `MySQL on EC2` | Tự cài, tự backup — không phải managed service |
| `DynamoDB` | NoSQL key-value, không phải MySQL |
| `Neptune` | Graph database, không phải MySQL |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Chọn managed database service
- **Constraints:** Phải tương thích MySQL + có automated backups

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- `Amazon Aurora` là cloud-native relational database của AWS, tương thích hoàn toàn với MySQL và PostgreSQL, hiệu năng gấp 5 lần MySQL thông thường.
- Aurora cung cấp **automated backups** hàng ngày với retention mặc định 7 ngày (có thể tăng lên tối đa 35 ngày), kèm **point-in-time recovery**.
- Ngoài ra còn có `automated failover` (Multi-AZ), `backtrack` (quay lại thời điểm trước đó), và `Aurora Global Database` cho DR đa Region.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `MySQL trên EC2` là self-managed — bạn tự cài đặt, patch, backup bằng cron/script. Đề hỏi về AWS database service có **automated backups**, EC2 install không phải.

**❌ Đáp án C:**
- `DynamoDB` là NoSQL key-value & document database, không tương thích MySQL. Có point-in-time recovery (PITR) riêng, nhưng không phải MySQL.

**❌ Đáp án D:**
- `Amazon Neptune` là fully-managed **graph database** (chuẩn Property Graph + RDF/SPARQL), không phải MySQL. Dùng cho mạng lưới quan hệ, social network, knowledge graph, ...

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"MySQL-compatible + managed + auto backup = `Aurora`. Graph = `Neptune`, NoSQL key-value = `DynamoDB`. MySQL trên EC2 = tự lo hết."*
