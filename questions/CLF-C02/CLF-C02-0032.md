# Câu hỏi #32

Which of the following are examples of AWS-Managed Services, where AWS is responsible for the operational and maintenance burdens of running the service? (Choose TWO)

## Options

**A.** Amazon VPC.
**B.** Amazon DynamoDB.
**C.** Amazon Elastic MapReduce.
**D.** AWS IAM.
**E.** Amazon Elastic Compute Cloud.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về `AWS-Managed Services` — dịch vụ mà AWS chịu phần lớn trách nhiệm vận hành.
- **Existing Resources:** Không có tài nguyên cụ thể.
- **Current Issue/Goal:** Xác định 2 dịch vụ điển hình thuộc nhóm AWS-managed.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `AWS-Managed Services` | AWS chịu operational/maintenance burden cho khách hàng |
| `Amazon DynamoDB` | NoSQL serverless, AWS quản lý patching, scaling, replication |
| `Amazon EMR` | Managed big data (Hadoop/Spark), AWS quản lý cluster |
| `Amazon VPC` | Khách hàng tự cấu hình network (subnet, route, ...) |
| `EC2` | IaaS — khách hàng lo OS, patching, app |
| `IAM` | Khách hàng quản lý user/policy |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Multi-answer (Choose TWO) — AWS-managed services
- **Constraints:** Phải là service AWS chịu vận hành hoàn toàn, không phải IaaS

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B và C**

**Giải thích:**
- **B — Amazon DynamoDB:** NoSQL database hoàn toàn được AWS quản lý — tự động scale, replicate across AZ, backup, patching, monitoring. Khách hàng chỉ tạo table và đọc/ghi dữ liệu, không cần quản lý server.
- **C — Amazon EMR (Elastic MapReduce):** Managed big data service cho Hadoop, Spark, Hive, Presto, ... AWS chịu trách nhiệm provision, cấu hình, patching và quản lý cluster. Khách hàng tập trung vào job và data analysis.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `Amazon VPC` cung cấp network boundary, nhưng khách hàng phải tự thiết kế subnet, route table, NAT, security group, ... — không phải hoàn toàn managed.

**❌ Đáp án D:**
- `AWS IAM` chỉ là dịch vụ quản lý identity, AWS không "vận hành" user của khách hàng. Khách hàng tự tạo user, group, role, policy.

**❌ Đáp án E:**
- `Amazon EC2` là IaaS — khách hàng hoàn toàn chịu trách nhiệm về OS, patching, runtime, app. Đây là ví dụ điển hình KHÔNG phải managed service.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Managed = AWS chịu vận hành (DynamoDB, EMR, RDS, Lambda, S3). IaaS = bạn lo (EC2, VPC)."*
