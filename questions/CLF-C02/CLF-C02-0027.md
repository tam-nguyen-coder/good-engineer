# Câu hỏi #27

A company is deploying a new two-tier web application in AWS. Where should the most frequently accessed data be stored so that the application's response time is optimal?

## Options

**A.** AWS OpsWorks.
**B.** AWS Storage Gateway.
**C.** Amazon EBS volume.
**D.** Amazon ElastiCache.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Công ty triển khai web application 2-tier (web + database). Cần tối ưu **response time** bằng cách lưu dữ liệu **truy cập thường xuyên** ở đâu.
- **Existing Resources:** 2-tier web application (web server + DB).
- **Current Issue/Goal:** Giảm latency, tăng tốc độ phản hồi cho dữ liệu hot (frequently accessed).

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `most frequently accessed data` | Dữ liệu "hot" — đọc nhiều |
| `optimal response time` | Cần giảm latency tối đa |
| `Amazon ElastiCache` | In-memory cache (Redis/Memcached), nhanh nhất |
| `EBS volume` | Block storage nhanh nhưng vẫn phải truy vấn DB |
| `Storage Gateway` | Hybrid storage, không phù hợp web app latency |
| `OpsWorks` | Configuration management, không phải data store |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance cho frequently accessed data
- **Constraints:** Cần tốc độ nhanh nhất có thể

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: D**

**Giải thích:**
- `Amazon ElastiCache` cung cấp in-memory cache (Redis hoặc Memcached) với latency cỡ **microsecond đến millisecond**, nhanh hơn rất nhiều so với truy vấn database truyền thống.
- Các kết quả truy vấn "hot" (ví dụ session data, product catalog, leader board) được cache trong RAM, application đọc trực tiếp từ cache thay vì query database.
- Hai pattern phổ biến: `Lazy Loading` (chỉ cache khi đọc) và `Write-Through` (ghi đồng thời vào cache + DB).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `AWS OpsWorks` là configuration management (Chef/Puppet), không phải data store.

**❌ Đáp án B:**
- `AWS Storage Gateway` là hybrid storage kết nối on-premises với AWS Cloud, thường dùng cho backup/archive, không tối ưu cho latency web app.

**❌ Đáp án C:**
- `EBS` là block storage gắn với EC2, có thể dùng làm data store cho DB. Tuy nhiên, để tối ưu response time cho frequently accessed data, cần lớp cache in-memory phía trước. EBS vẫn chậm hơn ElastiCache rất nhiều.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Dữ liệu hot + response time tối ưu = `ElastiCache` (in-memory). `EBS` cho block storage, `Storage Gateway` cho hybrid, `OpsWorks` cho config."*
