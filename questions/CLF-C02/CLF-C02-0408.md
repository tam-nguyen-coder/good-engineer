# Câu hỏi #408

Which task is AWS responsible for in the shared responsibility model for security and compliance?

## Options

**A.** Granting access to individuals and services.
**B.** Encrypting data in transit.
**C.** Updating Amazon EC2 host firmware.
**D.** Updating operating systems.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về phân chia trách nhiệm giữa AWS và khách hàng theo mô hình Shared Responsibility.
- **Existing Resources:** Không có cụ thể — bài toán khái niệm.
- **Current Issue/Goal:** Xác định tác vụ nào hoàn toàn thuộc về AWS (Security **of** the Cloud).

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `shared responsibility model` | AWS chịu security **of** the cloud, khách hàng chịu security **in** the cloud |
| `updating EC2 host firmware` | Thuộc về hạ tầng vật lý mà AWS quản lý |
| `encrypting data in transit` | Khách hàng có thể tự cấu hình TLS |
| `updating operating systems` | EC2 thuộc trách nhiệm khách hàng (IaaS) |
| `granting access` | Khách hàng quản lý IAM users/roles |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Phân biệt trách nhiệm AWS vs khách hàng
- **Constraints:** Câu hỏi về "AWS responsibility" (infrastructure layer)

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
- Theo Shared Responsibility Model, AWS chịu trách nhiệm về `Security of the Cloud`: hạ tầng vật lý, datacenter, server, host firmware, hypervisor, network và storage infrastructure.
- Cập nhật firmware của máy chủ vật lý (host firmware) là tác vụ hạ tầng mà AWS thực hiện, khách hàng không cần và cũng không thể can thiệp.
- Đây là điểm khác biệt quan trọng so với việc khách hàng tự quản lý OS ở trên EC2 (thuộc trách nhiệm khách hàng).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- "Granting access to individuals and services" là việc của khách hàng thông qua `IAM users`, `roles`, `policies`.
- Đáp án này thuộc `Security in the Cloud` — do khách hàng tự quản lý.

**❌ Đáp án B:**
- "Encrypting data in transit" (TLS/SSL) có thể do AWS cung cấp sẵn cho một số dịch vụ managed (ví dụ: S3, RDS), nhưng việc cấu hình và đảm bảo encryption đầu cuối thuộc trách nhiệm chung/riêng của khách hàng tùy ngữ cảnh.
- Câu hỏi Cloud Practitioner thường xem đây là **shared control** chứ không thuộc riêng AWS.

**❌ Đáp án D:**
- "Updating operating systems" trên EC2 thuộc trách nhiệm của khách hàng (vì EC2 là IaaS). Với managed services như RDS, AWS lo OS patching.
- Đáp án này chỉ đúng nếu dịch vụ cụ thể là managed (RDS, Lambda…) chứ không phải EC2 mặc định.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *AWS chịu: hạ tầng vật lý, `host firmware`, `hypervisor`, datacenter. Khách hàng chịu: `IAM`, `OS patching` trên EC2, encryption cấu hình.*
