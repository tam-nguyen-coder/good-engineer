# Câu hỏi #371

Engineers are wasting a lot of time and effort managing batch computing software in traditional data centers. Which of the following AWS services allows them to easily run thousands of batch computing jobs?

## Options

**A.** Amazon EC2.
**B.** AWS Batch.
**C.** Lambda@Edge.
**D.** AWS Fargate.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _Engineers are wasting a lot of time and effort managing batch computing software in traditional data centers. Which of t..._
- **Existing Resources:** Đề nhắc tới: `Batch` — chạy batch job quy mô lớn.
- **Current Issue/Goal:** Lựa chọn giải pháp có hiệu năng cao nhất, độ trễ thấp nhất.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `Hybrid / On-premises` | Kết nối môi trường on-premises với AWS |
| ``EC2`` | dịch vụ compute (máy chủ ảo) trên AWS Cloud |
| ``Lambda`` | dịch vụ serverless compute chạy theo sự kiện |
| ``Fargate`` | serverless compute cho container |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance / Lowest latency
- **Constraints:** Hiệu năng và độ trễ thấp

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
Đáp án `Batch` đúng vì chạy batch job quy mô lớn, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Batch` đóng vai trò: thực thi job tính toán mà không lo provisioning.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: `EC2` (dịch vụ compute (máy chủ ảo) trên AWS Cloud) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy ứng dụng, web server, batch processing được ưu tiên hơn hiệu năng.

**❌ Đáp án C:**
- Tại sao sai: `Lambda` (dịch vụ serverless compute chạy theo sự kiện) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy code mà không cần quản lý server, trả theo millisecond được ưu tiên hơn hiệu năng.

**❌ Đáp án D:**
- Tại sao sai: `Fargate` (serverless compute cho container) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu chạy container mà không cần quản lý EC2 được ưu tiên hơn hiệu năng.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **B** = `EC2`. Hiệu năng = edge location + cache + chọn instance/region gần user.*
