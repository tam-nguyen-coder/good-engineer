# Câu hỏi #529

A customer needs to run a MySQL database that easily scales. Which AWS service should they use?

## Options

**A.** Amazon Aurora.
**B.** Amazon Redshift.
**C.** Amazon DynamoDB.
**D.** Amazon ElastiCache.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _A customer needs to run a MySQL database that easily scales. Which AWS service should they use?_
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn giải pháp có hiệu năng cao nhất, độ trễ thấp nhất.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| ``DynamoDB`` | dịch vụ NoSQL key-value được quản lý hoàn toàn |
| ``Redshift`` | data warehouse columnar |
| ``Aurora`` | database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Best performance / Lowest latency
- **Constraints:** Hiệu năng và độ trễ thấp

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
Đáp án `Aurora` đúng vì database quan hệ tương thích MySQL/PostgreSQL hiệu năng cao, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Aurora` đóng vai trò: RDS cao cấp với storage tự scale, 6 bản sao.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- Tại sao sai: `Redshift` (data warehouse columnar) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu phân tích dữ liệu lớn, OLAP, BI được ưu tiên hơn hiệu năng.

**❌ Đáp án C:**
- Tại sao sai: `DynamoDB` (dịch vụ NoSQL key-value được quản lý hoàn toàn) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu lưu trữ dữ liệu phi quan hệ với độ trễ thấp ở mọi quy mô được ưu tiên hơn hiệu năng.

**❌ Đáp án D:**
- Tại sao sai: `ElastiCache` (in-memory cache (Redis, Memcached)) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng khi yêu cầu tăng tốc ứng dụng bằng cache, giảm tải database được ưu tiên hơn hiệu năng.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **A** = `DynamoDB`. Hiệu năng = edge location + cache + chọn instance/region gần user.*
