# Câu hỏi #419

Which of the following is a benefit of using the AWS Cloud?

## Options

**A.** Permissive security removes the administrative burden.
**B.** Ability to focus on revenue-generating activities.
**C.** Control over cloud network hardware.
**D.** Choice of specific cloud hardware vendors.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về lợi ích cốt lõi khi chuyển sang AWS Cloud.
- **Existing Resources:** Không có cụ thể — bài toán khái niệm.
- **Current Issue/Goal:** Xác định value proposition của AWS Cloud.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `benefit of AWS Cloud` | Lợi ích kinh doanh/chiến lược |
| `focus on revenue-generating activities` | IT trở thành enabler thay vì bottleneck |
| `permissive security` | Quan niệm sai — AWS yêu cầu khách hàng quản lý security in the cloud |
| `control over cloud network hardware` | Khách hàng không chọn được hardware cụ thể |
| `choice of specific cloud hardware vendors` | AWS không cho chọn hardware vendor (Intel/AMD/ARM) cho hầu hết services |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Lợi ích business
- **Constraints:** Câu hỏi single-answer, chọn đúng benefit

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- Một trong những lợi ích lớn nhất của cloud là giải phóng nguồn lực IT khỏi các hoạt động "undifferentiated heavy lifting" (vận hành hạ tầng, mua server, bảo trì datacenter…) để tập trung vào hoạt động tạo giá trị kinh doanh.
- Developer/IT team có thể tập trung vào sản phẩm, cải tiến trải nghiệm khách hàng, đổi mới — thay vì lo về capacity planning, hardware refresh…
- Đây là value proposition cốt lõi mà AWS quảng bá: focus on what differentiates your business, not on infrastructure.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- "Permissive security removes the administrative burden" là **sai về mặt khái niệm**. AWS yêu cầu khách hàng chịu trách nhiệm về security in the cloud (IAM, encryption, OS patching, network config…).
- Đáp án này đi ngược lại Shared Responsibility Model.

**❌ Đáp án C:**
- "Control over cloud network hardware" — AWS là public cloud, khách hàng không có quyền truy cập vật lý vào network hardware (router, switch, firewall appliance…).
- Đáp án này phù hợp với on-premises private data center.

**❌ Đáp án D:**
- "Choice of specific cloud hardware vendors" — AWS tự chọn CPU vendor cho các instance type. Khách hàng chỉ chọn instance type/family, không chọn được Intel vs AMD vs ARM cụ thể cho hầu hết services.
- Đáp án này cũng không đúng với mô hình public cloud.

## 6. MẺO GHI NHỚ (Memory Hook)
🧠 *AWS = tập trung business khác biệt. AWS không cho bạn chọn hardware vendor hay sờ network switch. Bạn vẫn chịu trách nhiệm security in the cloud.*
