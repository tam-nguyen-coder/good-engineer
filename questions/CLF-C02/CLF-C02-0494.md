# Câu hỏi #494

Which AWS service can be used to automatically scale an application up and down without making capacity planning decisions?

## Options

**A.** Amazon EBS.
**B.** Amazon Redshift.
**C.** AWS CloudTrail.
**D.** AWS Lambda.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Ngữ cảnh đề bài: _Which AWS service can be used to automatically scale an application up and down without making capacity planning decisio..._
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Cần giám sát, theo dõi và ghi nhận hoạt động của hệ thống/tài khoản.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| ``Lambda`` | dịch vụ serverless compute chạy theo sự kiện |
| ``CloudTrail`` | dịch vụ audit API call |
| ``EBS`` | Elastic Block Store - ổ đĩa block cho EC2 |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Monitoring / Auditing
- **Constraints:** Tuân theo best practice AWS

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: D**

**Giải thích:**
Đáp án `Lambda` đúng vì dịch vụ serverless compute chạy theo sự kiện, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `Lambda` đóng vai trò: chạy code mà không cần quản lý server, trả theo millisecond.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- Tại sao sai: `EBS` (Elastic Block Store - ổ đĩa block cho EC2) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `EBS`.

**❌ Đáp án B:**
- Tại sao sai: `Redshift` (data warehouse columnar) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `Redshift`.

**❌ Đáp án C:**
- Tại sao sai: `CloudTrail` (dịch vụ audit API call) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `CloudTrail`.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Đáp án **D** = `Lambda` — phù hợp với yêu cầu đề bài.*
