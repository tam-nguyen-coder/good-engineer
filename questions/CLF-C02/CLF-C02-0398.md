# Câu hỏi #398

Which AWS service can be used to manually launch instances based on resource requirements?

## Options

**A.** Amazon EBS.
**B.** Amazon S3.
**C.** Amazon EC2.
**D.** Amazon ECS.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Đề hỏi về khái niệm/dịch vụ AWS liên quan tới: _Which AWS service can be used to manually launch instances based on resource req..._
- **Existing Resources:** Không có tài nguyên cụ thể — câu hỏi thuộc dạng lý thuyết/khái niệm.
- **Current Issue/Goal:** Lựa chọn đáp án đúng nhất theo yêu cầu của đề.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| ``EC2`` | dịch vụ compute (máy chủ ảo) trên AWS Cloud |
| ``S3`` | dịch vụ lưu trữ object bền vững (11x 9s) |
| ``ECS`` | Elastic Container Service |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Compute
- **Constraints:** Tuân theo best practice AWS

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
Đáp án `EC2` đúng vì dịch vụ compute (máy chủ ảo) trên AWS Cloud, phù hợp với yêu cầu của đề.
Trong ngữ cảnh này, `EC2` đóng vai trò: chạy ứng dụng, web server, batch processing.

**❌ Đáp án A:**
- Tại sao sai: `EBS` (Elastic Block Store - ổ đĩa block cho EC2) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `EBS`.

**❌ Đáp án B:**
- Tại sao sai: `S3` (dịch vụ lưu trữ object bền vững (11x 9s)) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `S3`.

**❌ Đáp án D:**
- Tại sao sai: `ECS` (Elastic Container Service) không phù hợp với yêu cầu cụ thể của đề.
- Khi nào đúng: Đáp án này sẽ đúng trong tình huống khác yêu cầu cụ thể `ECS`.
