# Câu hỏi #413

Which AWS service identifies security groups that allow unrestricted access to a user's AWS resources?

## Options

**A.** AWS Trusted Advisor.
**B.** Amazon Inspector.
**C.** Amazon CloudWatch.
**D.** AWS CloudTrail.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi về dịch vụ tự động phát hiện security group có rule "open to the world" (0.0.0.0/0 hoặc ::/0) gây rủi ro bảo mật.
- **Existing Resources:** Một số security group trong AWS environment.
- **Current Issue/Goal:** Chủ động tìm ra security group mở không giới hạn.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `identifies security groups` | Câu hỏi về security posture assessment |
| `unrestricted access` | SG rule mở 0.0.0.0/0 hoặc ::/0 |
| `AWS Trusted Advisor` | Dịch vụ kiểm tra best practices, có check "Security Groups - Unrestricted Access" |
| `Amazon Inspector` | Đánh giá lỗ hổng EC2/container, không focus vào SG |
| `Amazon CloudWatch` | Monitoring metrics/logs, không check SG |
| `AWS CloudTrail` | Audit API calls, không check SG configuration |

## 3. YÊU CẦU CỀ
- **Question type:** Xác định dịch vụ phát hiện SG mở
- **Constraints:** Cần cảnh báo/proactive check về SG rule

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: A**

**Giải thích:**
- `AWS Trusted Advisor` cung cấp check "Security Groups - Specific Ports Unrestricted" và "Security Groups - Unrestricted Access" tự động phát hiện các security group mở cổng ra internet công cộng (ví dụ: SSH 22, RDP 3389, HTTP 80).
- Trusted Advisor còn cung cấp các check khác thuộc 5 categories: Cost Optimization, Performance, Security, Fault Tolerance, Service Limits.
- Với `Business` và `Enterprise` Support plan, bạn có đầy đủ các check; với `Basic`/`Developer`, chỉ một số check miễn phí.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án B:**
- `Amazon Inspector` đánh giá lỗ hổng trên EC2 instances và container images (CVE, networking reachability), tập trung vào OS/application vulnerabilities chứ không phải SG rule check.
- Đáp án này dùng cho security assessment runtime.

**❌ Đáp án C:**
- `Amazon CloudWatch` theo dõi metrics, logs, events. Không có check cụ thể về SG unrestricted access (mặc dù có thể dựng custom rule qua `Config Rules`).
- Đáp án này dùng cho monitoring.

**❌ Đáp án D:**
- `AWS CloudTrail` ghi lại API calls (ví dụ: ai đã tạo/sửa SG), giúp audit nhưng không chủ động phát hiện SG mở.
- Đáp án này dùng cho audit trail.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *SG mở không giới hạn = `Trusted Advisor` check. Muốn custom rule tự động enforce: dùng `AWS Config Rules`.*
