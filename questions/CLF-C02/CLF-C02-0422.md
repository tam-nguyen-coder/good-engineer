# Câu hỏi #422

Under the AWS shared responsibility model, which of the following activities are the customer's responsibility? (Select TWO)

## Options

**A.** Patching operating system components for Amazon Relational Database Server (Amazon RDS).
**B.** Encrypting data on the client-side.
**C.** Training the data center staff.
**D.** Configuring Network Access Control Lists (ACL).
**E.** Maintaining environmental controls within a data center.


---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Câu hỏi multi-answer về trách nhiệm **của khách hàng** trong Shared Responsibility Model.
- **Existing Resources:** Không có — bài toán khái niệm.
- **Current Issue/Goal:** Chọn 2 hoạt động thuộc trách nhiệm khách hàng (Security in the cloud).

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `customer's responsibility` | Security **in** the cloud |
| `Encrypting data on the client-side` | Khách hàng tự mã hóa trước khi gửi lên AWS |
| `Configuring Network ACLs` | Khách hàng cấu hình NACL trong VPC của mình |
| `Patching OS for RDS` | AWS chịu (RDS là managed) |
| `Training data center staff` | AWS chịu (datacenter của AWS) |
| `Environmental controls` | AWS chịu (physical infrastructure) |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Khái niệm Shared Responsibility (multi-answer)
- **Constraints:** Phải chọn 2 sole-customer activities

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B và D**

**Giải thích:**
- `Encrypting data on the client-side` (client-side encryption): khách hàng tự mã hóa dữ liệu trước khi upload lên AWS, giữ control hoàn toàn với key. Thuộc trách nhiệm khách hàng.
- `Configuring Network Access Control Lists (NACL)`: NACL là stateless firewall ở subnet level trong VPC, thuộc quyền cấu hình của khách hàng.
- Cả hai đều là hoạt động Security **in** the cloud, do khách hàng thực hiện.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- `Amazon RDS` là managed service, AWS tự động patch OS cho customer. Khách hàng **không có quyền** patch OS cho RDS instance.
- Đáp án này là sole-AWS cho managed services. Với EC2 thì khách hàng mới chịu OS patching.

**❌ Đáp án C:**
- "Training data center staff" thuộc trách nhiệm của AWS — đây là nhân viên vận hành datacenter của AWS.
- Đáp án này thuộc Security **of** the Cloud.

**❌ Đáp án E:**
- "Environmental controls" (làm mát, phòng cháy chữa cháy, nguồn điện) trong data center thuộc trách nhiệm AWS.
- Đáp án này thuộc Security **of** the Cloud.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Customer chịu: client-side encryption, NACL/SG config, IAM, data, OS trên EC2. AWS chịu: physical, environmental, RDS OS patching, data center staff.*
