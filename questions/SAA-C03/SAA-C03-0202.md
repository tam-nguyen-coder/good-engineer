# Question #202 - Topic 1

A company is planning to move its data to an Amazon S3 bucket. The data must be encrypted when it is stored in the S3 bucket. Additionally, the encryption key must be automatically rotated every year. Which solution will meet these requirements with the LEAST operational overhead?

## Options

**A.** Move the data to the S3 bucket. Use server-side encryption with Amazon S3 managed encryption keys (SSE-S3). Use the built-in key rotation behavior of SSE-S3 encryption keys.

**B.** Create an AWS Key Management Service (AWS KMS) customer managed key. Enable automatic key rotation. Set the S3 bucket's default encryption behavior to use the customer managed KMS key. Move the data to the S3 bucket.

**C.** Create an AWS Key Management Service (AWS KMS) customer managed key. Set the S3 bucket's default encryption behavior to use the customer managed KMS key. Move the data to the S3 bucket. Manually rotate the KMS key every year.

**D.** Encrypt the data with customer key material before moving the data to the S3 bucket. Create an AWS Key Management Service (AWS KMS) key without key material. Import the customer key material into the KMS key. Enable automatic key rotation.

---

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Data → S3, encrypt at rest, auto rotate key yearly.
- **Existing Resources:** None.
- **Current Issue/Goal:** Encryption + key rotation, least overhead.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `automatically rotated every year` | KMS Customer Managed Key (CMK) + Auto Key Rotation (xoay vòng sau mỗi 365 ngày) |
| `least operational overhead` | Chọn giải pháp KMS CMK tự động hóa thay vì xoay vòng thủ công |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Security / Encryption
- **Constraints:** Encrypt at rest, auto key rotation every year, least overhead

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- **KMS Customer Managed Key (CMK) + Auto Rotation:** Cho phép tạo một khóa KMS do khách hàng quản lý và bật tính năng xoay vòng khóa tự động (KMS sẽ tự động xoay vòng key material sau mỗi 365 ngày). Cấu hình này chỉ cần thiết lập một lần trên S3 bucket (default encryption), giúp tự động hóa hoàn toàn và đáp ứng chính xác yêu cầu xoay vòng hàng năm với operational overhead thấp nhất.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- **SSE-S3 (S3 managed keys):** Amazon S3 tự động xoay vòng các master key của nó thường xuyên, tuy nhiên AWS không cam kết tần suất xoay vòng cụ thể là hàng năm (every year) và khách hàng không thể kiểm soát hay kiểm toán (audit) lịch trình xoay vòng này. Do đó, SSE-S3 không đáp ứng yêu cầu tuân thủ cụ thể của đề bài.

**❌ Đáp án C:**
- **Manual rotation:** Xoay vòng khóa thủ công hàng năm yêu cầu quản trị viên phải tự thực hiện định kỳ, làm tăng operational overhead đáng kể và dễ xảy ra sai sót.

**❌ Đáp án D:**
- **Imported key material:** Khóa KMS được import key material từ bên ngoài không hỗ trợ tính năng tự động xoay vòng khóa (automatic key rotation) của KMS. Bạn bắt buộc phải tạo key version mới và import material thủ công.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Yêu cầu rotate key định kỳ (every year) -> Phải dùng KMS Customer Managed Key bật Auto Rotation. SSE-S3 tự động rotate nhưng không có lịch trình cam kết/thiết lập cụ thể cho khách hàng."*
