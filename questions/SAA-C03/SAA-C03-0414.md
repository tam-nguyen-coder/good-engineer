# Question #414 - Topic 1

A company has a business system that generates hundreds of reports each day. The business system saves the reports to a network share in CSV format. The company needs to store this data in the AWS Cloud in near-real time for analysis. Which solution will meet these requirements with the LEAST administrative overhead?

## Options

**A.** Use AWS DataSync to transfer the files to Amazon S3. Create a scheduled task that runs at the end of each day.

**B.** Create an Amazon S3 File Gateway. Update the business system to use a new network share from the S3 File Gateway.

**C.** Use AWS DataSync to transfer the files to Amazon S3. Create an application that uses the DataSync API in the automation workflow.

**D.** Deploy an AWS Transfer for SFTP endpoint. Create a script that checks for new files on the network share and uploads the new files by using SFTP.

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Business system writes CSV reports to network share. Need near-real time storage in AWS for analysis.
- **Existing Resources:** On-prem business system, network share.
- **Current Issue/Goal:** Near-real time S3 upload. Least admin overhead.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `near-real time` | Sync liên tục, không batch cuối ngày. |
| `least administrative overhead` | Minimal config, managed service. |
| `S3 File Gateway` | Mount S3 as network share (NFS/SMB) → tự động upload. |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Least admin overhead / Storage
- **Constraints:** Near-real time, CSV on network share

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**

**Giải thích:**
- **S3 File Gateway (AWS Storage Gateway):** Cho phép gắn kết (mount) một Amazon S3 bucket dưới dạng một thư mục chia sẻ mạng cục bộ (NFS hoặc SMB). Đối với hệ thống nghiệp vụ (business system), việc lưu trữ báo cáo CSV chỉ cần thay đổi đường dẫn lưu trữ từ share mạng cũ sang share mạng mới do S3 File Gateway cung cấp.
- **Tải lên Near-real time:** Bất kỳ tệp CSV nào khi được ghi vào share mạng này sẽ được lưu tạm ở bộ đệm local cache của gateway và tự động đẩy bất đồng bộ lên Amazon S3 gần như ngay lập tức (near-real time).
- **Least Administrative Overhead:** Đây là giải pháp hoàn toàn managed từ AWS. Doanh nghiệp không cần viết hay bảo trì bất kỳ dòng script nào, không cần cấu hình task scheduler, giảm thiểu tối đa công sức quản trị.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- **DataSync chạy cuối ngày (End of day):** Việc lập lịch chạy vào cuối mỗi ngày biến tiến trình thành xử lý hàng loạt (batch processing), không đáp ứng được yêu cầu truyền dữ liệu theo thời gian thực (near-real time).

**❌ Đáp án C:**
- **Tự viết ứng dụng gọi DataSync API:** Mặc dù đáp ứng được yếu tố real-time bằng cách gọi API mỗi khi có file mới, nhưng giải pháp này yêu cầu đội ngũ kỹ sư phải phát triển, kiểm thử và duy trì một ứng dụng custom tích hợp API. Điều này làm tăng đáng kể gánh nặng quản trị (administrative overhead) so với giải pháp cắm-là-chạy của S3 File Gateway.

**❌ Đáp án D:**
- **AWS Transfer for SFTP + Upload Script:** Đòi hỏi phải triển khai endpoint SFTP (tốn chi phí duy trì cao) và phát triển các script giám sát thư mục chia sẻ mạng cục bộ để phát hiện tệp mới rồi upload qua SFTP. Việc tự duy trì các đoạn script này tạo ra nhiều overhead vận hành và rủi ro gián đoạn khi script gặp lỗi.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Ghi file local lên S3 near-real time không muốn code script → Dùng S3 File Gateway (mount NFS/SMB)."*
