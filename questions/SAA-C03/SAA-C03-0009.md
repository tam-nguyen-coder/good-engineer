# Question #9 - Topic 1

A company is running an SMB file server in its data center. The file server stores large files that are accessed frequently for the first few days after the files are created. After 7 days the files are rarely accessed. The total data size is increasing and is close to the company's total storage capacity. A solutions architect must increase the company's available storage space without losing low-latency access to the most recently accessed files. The solutions architect must also provide file lifecycle management to avoid future storage issues. Which solution will meet these requirements?

## Options

**A.** Use AWS DataSync to copy data that is older than 7 days from the SMB file server to AWS.

**B.** Create an Amazon S3 File Gateway to extend the company's storage space. Create an S3 Lifecycle policy to transition the data to S3 Glacier Deep Archive after 7 days.

**C.** Create an Amazon FSx for Windows File Server file system to extend the company's storage space.

**D.** Install a utility on each user's computer to access Amazon S3. Create an S3 Lifecycle policy to transition the data to S3 Glacier Flexible Retrieval after 7 days.



## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Một công ty đang chạy một SMB file server tại trung tâm dữ liệu (on-premises). File server lưu trữ các file lớn, được truy cập thường xuyên trong vài ngày đầu sau khi tạo, sau 7 ngày thì hiếm khi được truy cập.
- **Existing Resources:** Dung lượng lưu trữ cục bộ sắp cạn kiệt do lượng dữ liệu tăng liên tục.
- **Current Issue/Goal:** Cần mở rộng dung lượng lưu trữ nhưng phải đảm bảo:
  1. Giữ nguyên truy cập độ trễ thấp (low-latency access) cho các file mới/truy cập gần đây.
  2. Tự động quản lý vòng đời (lifecycle management) để giải phóng không gian lưu trữ và tối ưu hóa chi phí.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa |
|---------|---------|
| `SMB file server` | Giao thức chia sẻ file qua mạng LAN (Server Message Block) dùng cho Windows/Linux clients |
| `accessed frequently for the first few days ... after 7 days rarely accessed` | Dữ liệu nóng (hot data) trong 7 ngày đầu, sau đó chuyển thành dữ liệu lạnh (cold data) |
| `without losing low-latency access` | Cần một cơ chế bộ nhớ đệm cục bộ (local cache) để lưu trữ dữ liệu nóng tại on-premises |
| `file lifecycle management` | Cần cơ chế tự động chuyển dữ liệu cũ sang kho lưu trữ rẻ hơn (như Glacier) |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Thiết kế hệ thống lưu trữ hỗn hợp (Hybrid Storage & Data Tiering)
- **Constraints:**
  - Hỗ trợ giao thức SMB hiện tại của công ty.
  - Cung cấp bộ đệm cục bộ để đảm bảo tốc độ truy cập nhanh cho dữ liệu mới.
  - Tự động chuyển đổi dữ liệu ít dùng (> 7 ngày) sang kho lưu trữ có chi phí rẻ hơn (archiving).

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B**
**Giải thích:**
- `Amazon S3 File Gateway` (một phần của AWS Storage Gateway) cho phép kết nối hạ tầng on-premises với Amazon S3 qua giao thức file tiêu chuẩn như SMB. Nó cung cấp một local cache (bộ nhớ đệm cục bộ) ngay tại data center để lưu trữ các file được truy cập gần đây nhất, đáp ứng yêu cầu **truy cập độ trễ thấp**.
- S3 File Gateway ghi dữ liệu trực tiếp vào một S3 bucket. Tại đây, ta có thể cấu hình `S3 Lifecycle policy` để tự động chuyển tiếp (transition) các file cũ hơn 7 ngày sang `S3 Glacier Deep Archive` (lớp lưu trữ rẻ nhất của AWS), giúp giải quyết triệt để bài toán dung lượng và tối ưu hóa chi phí.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:** `AWS DataSync` là công cụ dùng để di chuyển hoặc sao chép dữ liệu định kỳ, nó **không mở rộng dung lượng lưu trữ cục bộ** một cách liền mạch hay cung cấp cơ chế local cache cho người dùng SMB. Người dùng vẫn phải truy cập thủ công hoặc thay đổi đường dẫn lưu trữ.

**❌ Đáp án C:** `Amazon FSx for Windows File Server` là một hệ thống file SMB được quản lý hoàn toàn trên AWS. Tuy nhiên, việc tạo một FSx file system thuần túy trên AWS không tích hợp sẵn cơ chế local cache tại on-premises data center để duy trì low-latency cho người dùng cục bộ một cách tối ưu như Storage Gateway, đồng thời không sử dụng trực tiếp S3 Lifecycle policies sang Glacier Deep Archive để lưu trữ giá rẻ.

**❌ Đáp án D:** Việc cài đặt tiện ích/phần mềm (utility) trên máy tính của từng người dùng để truy cập Amazon S3 trực tiếp gây ra gánh nặng lớn về quản lý và vận hành (operational overhead), đồng thời phá vỡ mô hình SMB file server hiện có. Ngoài ra, việc dùng `S3 Glacier Flexible Retrieval` sẽ tốn kém hơn so với `S3 Glacier Deep Archive` cho những dữ liệu cực kỳ hiếm khi truy cập.

## 6. MẸO GHI NHỚ
🧠 *Khi đề bài yêu cầu: File Server dùng giao thức SMB/NFS ở on-premises + Thiếu dung lượng + Yêu cầu độ trễ thấp cho file mới (Local Cache) + Lưu trữ dữ liệu cũ giá rẻ lên Cloud → Chọn ngay `Amazon S3 File Gateway` kết hợp `S3 Lifecycle Policy`.*
🧠 *`S3 Glacier Deep Archive` là lựa chọn có chi phí lưu trữ rẻ nhất của AWS cho dữ liệu lưu trữ dài hạn và hiếm khi truy cập.*
🧠 *Tránh các giải pháp cài đặt ứng dụng trên máy khách (client utility) vì nó làm tăng overhead vận hành và không scale tốt.*
