# Question #455 - Topic 1

A company uses AWS Organizations. The company wants to operate some of its AWS accounts with different budgets. The company wants to receive alerts and automatically prevent provisioning of additional resources on AWS accounts when the allocated budget threshold is met during a specific period. Which combination of solutions will meet these requirements? (Choose three.)

## Options

**A.** Use AWS Budgets to create a budget. Set the budget amount under the Cost and Usage Reports section of the required AWS accounts.

**B.** Use AWS Budgets to create a budget. Set the budget amount under the Billing dashboards of the required AWS accounts.

**C.** Create an IAM user for AWS Budgets to run budget actions with the required permissions.

**D.** Create an IAM role for AWS Budgets to run budget actions with the required permissions.

**E.** Add an alert to notify the company when each account meets its budget threshold. Add a budget action that selects the IAM identity created with the appropriate config rule to prevent provisioning of additional resources.

**F.** Add an alert to notify the company when each account meets its budget threshold. Add a budget action that selects the IAM identity created with the appropriate service control policy (SCP) to prevent provisioning of additional resources.

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** AWS Organizations, different budgets per account. Alert + auto-prevent provisioning when threshold met.
- **Existing Resources:** AWS Organizations, multiple accounts.
- **Current Issue/Goal:** Budget alerts + automatic resource prevention.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `budget threshold` | AWS Budgets: set budget amount. |
| `prevent provisioning` | Budget action with SCP (service control policy). |
| `IAM role` | Budget actions need IAM role (not user). |
| `Choose three` | 3 answers required. |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Cost management / Governance
- **Constraints:** Budget alerts + auto prevention

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: B, D, F**

**Giải thích:**
- **B (Thiết lập ngân sách):** AWS Budgets được thiết lập và quản lý thông qua giao diện **Billing Dashboard** (Bảng điều khiển thanh toán) của tài khoản AWS. Đây là nơi cấu hình số tiền ngân sách, chu kỳ theo dõi (tháng, quý, năm) và các ngưỡng cảnh báo.
- **D (Cấp quyền thực thi bằng IAM Role):** Tính năng **AWS Budgets Actions** cho phép tự động thực hiện các hành động ứng phó khi ngân sách vượt ngưỡng. Để AWS Budgets có thể thực thi các hành động này (như áp dụng SCP), dịch vụ AWS Budgets cần giả lập (assume) một **IAM Role** được cấu hình chính sách ủy quyền tin cậy (trust policy) và các quyền cần thiết. AWS Budgets không thể sử dụng IAM User để thực thi tự động.
- **F (Tự động ngăn chặn bằng SCP):** Khi vượt ngưỡng ngân sách, hành động tự động (**Budget Action**) sẽ áp dụng một **Service Control Policy (SCP)** lên tài khoản AWS đó trong AWS Organizations. SCP này sẽ từ chối quyền khởi tạo các tài nguyên mới (ví dụ: `Deny ec2:RunInstances`, `Deny rds:CreateDBInstance`), từ đó ngăn chặn phát sinh thêm chi phí hiệu quả.

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- AWS Budgets không được cấu hình dưới mục **Cost and Usage Reports** (Báo cáo chi phí và sử dụng - vốn là công cụ xuất dữ liệu chi tiết ra S3). Ngân sách phải được thiết lập trong mục Billing Dashboard.

**❌ Đáp án C:**
- Dịch vụ AWS Budgets thực thi các hành động tự động thông qua cơ chế uỷ thác quyền bằng **IAM Role** (role-based delegation), chứ không thể giả lập hay đăng nhập bằng tài khoản **IAM User** (chứa thông tin đăng nhập cá nhân).

**❌ Đáp án E:**
- **AWS Config rules:** Là công cụ dùng để đánh giá tính tuân thủ cấu hình của tài nguyên (detective control) chứ không có khả năng ngăn chặn trực tiếp việc khởi tạo tài nguyên mới (preventive control) tại thời điểm yêu cầu được gửi tới AWS API. Ngược lại, **SCP** trong Organizations là giải pháp lý tưởng nhất để chặn quyền thực thi ngay từ gốc.
