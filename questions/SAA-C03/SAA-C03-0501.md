# Question #501 - Topic 1

A company wants to ingest customer payment data into the company's data lake in Amazon S3. The company receives payment data every minute on average. The company wants to analyze the payment data in real time. Then the company wants to ingest the data into the data lake. Which solution will meet these requirements with the MOST operational efficiency?

## Options

**A.** Use Amazon Kinesis Data Streams to ingest data. Use AWS Lambda to analyze the data in real time.

**B.** Use AWS Glue to ingest data. Use Amazon Kinesis Data Analytics to analyze the data in real time.

**C.** Use Amazon Kinesis Data Firehose to ingest data. Use Amazon Kinesis Data Analytics to analyze the data in real time.

**D.** Use Amazon API Gateway to ingest data. Use AWS Lambda to analyze the data in real time.

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Công ty cần ingest dữ liệu thanh toán (dạng streaming, mỗi phút một lần) vào data lake S3. Yêu cầu: phân tích real-time trước, sau đó mới đưa vào S3.
- **Existing Resources:** Data Lake trên S3, nguồn dữ liệu payment.
- **Current Issue/Goal:** Cần pipeline real-time vừa phân tích vừa lưu trữ, operational efficiency cao nhất.

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `ingest data into Amazon S3 data lake` | Cần service tự động ghi dữ liệu xuống S3. |
| `analyze in real time` | Cần real-time analytics, không phải batch. |
| `MOST operational efficiency` | Càng ít phải tự quản lý càng tốt (serverless/managed). |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Most operational efficiency
- **Constraints:** Real-time analysis + deliver to S3 data lake

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: C**

**Giải thích:**
- **Lưu ý quan trọng về Rebranding của AWS:** 
  - **Amazon Kinesis Data Analytics (KDA)** đã được AWS đổi tên thành **Amazon Managed Service for Apache Flink** vào tháng 8 năm 2023.
  - **Amazon Kinesis Data Firehose (KDF)** cũng được đổi tên thành **Amazon Data Firehose (ADF)** vào đầu năm 2024.
  - *Tuy nhiên, đề thi SAA-C03 vẫn giữ nguyên các tên gọi cũ (KDA, KDF) trong các câu hỏi cũ. Do đó, bạn cần ghi nhớ cả tên gọi cũ và mới.*
- **Phân tích real-time và đẩy về S3:** 
  - **Amazon Data Firehose (Kinesis Data Firehose)** là dịch vụ fully managed chuyên dùng để truyền tải dữ liệu streaming (ingestion) trực tiếp tới các đích lưu trữ như Amazon S3 data lake mà không cần quản lý hạ tầng hay tự viết code consumer để ghi file.
  - **Managed Service for Apache Flink (Kinesis Data Analytics)** cho phép đọc dữ liệu trực tiếp từ Firehose/Kinesis Stream, thực hiện các truy vấn phân tích dữ liệu thời gian thực (real-time SQL hoặc Apache Flink application) và trả lại kết quả đầu ra cho Firehose để tự động đẩy vào S3. Sự kết hợp này mang lại giải pháp hoàn chỉnh từ phân tích đến lưu trữ với gánh nặng vận hành tối thiểu (**MOST operational efficiency**).

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án A:**
- **Kinesis Data Streams (KDS) + Lambda:** KDS là dịch vụ streaming thô (raw data stream). Để xử lý dữ liệu và đẩy về S3, bạn phải tự viết code cho Lambda làm consumer, xử lý việc chia nhỏ dữ liệu và thực thi logic ghi dữ liệu vào S3. Việc này đòi hỏi nhiều công sức lập trình và quản lý (operational overhead) hơn so với việc dùng Firehose có sẵn tính năng tích hợp ghi S3 tự động.

**❌ Đáp án B:**
- **AWS Glue:** Là dịch vụ ETL được thiết kế tối ưu cho việc xử lý hàng loạt (batch processing/ETL) theo lịch trình, không phù hợp cho việc phân tích và ingest thời gian thực (real-time stream).

**❌ Đáp án D:**
- **API Gateway + Lambda:** API Gateway thiết kế cho REST/HTTP API và request/response đồng bộ. Việc sử dụng nó để ingest luồng dữ liệu liên tục theo phút sẽ không tối ưu hiệu năng và chi phí bằng dịch vụ chuyên dụng cho streaming như Kinesis/Firehose.

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *"Phân tích real-time + Lưu S3 tự động → Amazon Data Firehose + Managed Service for Apache Flink (KDA cũ). Glue = Batch, API Gateway = REST, KDS = cần code tự ghi S3."*
