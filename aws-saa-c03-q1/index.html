A company collects 500 gigabytes of data daily from sites across multiple continents. Each site has high-speed internet. The goal is to aggregate all data as quickly as possible into a single Amazon S3 bucket with minimum operational complexity. Which solution meets these requirements?

Proposed Options
A
Turn on S3 Transfer Acceleration on the destination S3 bucket. Use multipart uploads to directly upload site data to the destination S3 bucket.
B
Upload data from each site to an S3 bucket in the closest Region. Use Cross-Region Replication to copy objects to the destination S3 bucket. Then remove data from the origin S3 bucket.
C
Schedule AWS Snowball Edge Storage Optimized device jobs daily to transfer data from each site to the closest Region. Use Cross-Region Replication to copy objects to the destination S3 bucket.
D
Upload data to an EC2 instance in the closest Region. Store data in an EBS volume, take EBS snapshots at regular intervals, and copy them to the destination Region.
ANSWER DEEP DIVE Detailed justifications
Technical Deep Dive
A
CORRECT ANSWER

Turn on S3 Transfer Acceleration and use multipart uploads...

S3TA: Routes data over private AWS private backbone via edge locations, bypassing public internet.
Multipart Uploads: Parallelizes transfers of large files (500 GB), maximizing throughput.
Serverless: Configured on S3 directly. Zero active servers, minimizing complexity.
B
INCORRECT

Upload to local S3 and use Cross-Region Replication (CRR)...

High overhead: Requires configuring multiple regional staging buckets.
Complexity: Requires custom logic scripts to delete source objects after copying completes.
C
INCORRECT

Schedule daily AWS Snowball Edge device jobs...

Logistical issue: Shipping, recycling, and provisioning physical devices daily is slow.
Unnecessary: Snowball is designed for one-time migrations, not daily 500 GB internet-connected streams.
D
INCORRECT

Upload to EC2, take EBS snapshots, copy snapshots...

Over-engineered: Introduces virtual servers, storage volumes, and snapshot backup policies.
High Maintenance: Requires active server monitoring, scripting copies, and increases costs.

Key Takeaways

AWS S3 Golden Rule:
Fast Network + Large Files + Distance =
S3TA + Multipart Uploads
