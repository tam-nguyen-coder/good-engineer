A company needs the ability to analyze the log files of its proprietary application. The logs are stored in JSON format in an Amazon S3 bucket. Queries will be simple and will run on-demand. A solutions architect needs to perform the analysis with minimal changes to the existing architecture. What should the solutions architect do to meet these requirements with the LEAST amount of operational overhead?

ANSWER DEEP DIVE Detailed justifications
Technical Deep Dive
C
CORRECT ANSWER

Use Amazon Athena directly with Amazon S3...

Serverless SQL: Athena runs standard SQL directly against S3 data lakes with zero servers to manage.
Minimal Changes: Keep data original in S3 format without performing ETL or changing ingestion flows.
Least Overhead: Zero infrastructure maintenance; you are only charged per query based on data scanned.
A
INCORRECT

Use Amazon Redshift to load all content...

Data Migration: Redshift requires provisioning a warehouse and copying S3 logs into database tables.
High Overhead: Demands ETL pipeline orchestration, database schema updates, and active cluster management.
B
INCORRECT

Use Amazon CloudWatch Logs to store the logs...

Architecture Change: Requires re-routing existing application logs from S3 to CloudWatch Logs.
Over-engineered: Moving terabytes of logs to CloudWatch is costly and violates the "minimal changes" design rule.
D
INCORRECT

AWS Glue catalog + transient Apache Spark on EMR...

Heavy Infrastructure: Managing custom EMR clusters requires runtime configurations and sizing.
Slow Startup: Spin up delays on Spark/EMR make on-demand query latency too high for simple analysis.
