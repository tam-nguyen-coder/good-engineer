ANSWER DEEP DIVE Detailed justifications
Technical Deep Dive
D
CORRECT ANSWER

Adding more EC2 instances of the same size to handle traffic

Scale Out: Adds identical instances to distribute traffic load.
High Availability: Integrates with Auto Scaling and Elastic Load Balancer.
Cloud Best Practice: Easily scales out and in to optimize resource costs.
A
INCORRECT

Replacing an existing instance with a larger one

Vertical Scaling: Upgrades a single node's configuration (Scale Up).
Not Horizontal: Replaces the resource rather than adding new ones.
B
INCORRECT

Increasing compute capacity of a single instance

Single Instance Limit: Modifies CPU/RAM properties of a single server.
Scale Up: Limited by physical hardware constraints of a single host.
C
INCORRECT

Adding more RAM capacity to an EC2 instance

Hardware Upgrade: Enhances memory of a single instance.
Vertical Scaling: Upgrades existing resources (no new nodes).
