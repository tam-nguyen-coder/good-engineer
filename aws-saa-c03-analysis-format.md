# AWS SAA C03 – Question Analysis Format

## 1. CONTEXT & ĐỀ BÀI
- **Scenario:** Tóm tắt tình huống đề cho (hệ thống gì, bài toán gì)
- **Existing Resources:** Những gì đã có sẵn (EC2, S3, RDS, ALB, v.v.)
- **Current Issue/Goal:** Vấn đề hiện tại hoặc mục tiêu cần đạt được

## 2. KEYWORDS QUAN TRỌNG
| Keyword | Ý nghĩa / Gợi ý |
|---------|-----------------|
| `keyword 1` | ... |
| `keyword 2` | ... |

## 3. YÊU CẦU CỦA ĐỀ
- **Question type:** Most cost-effective / Most secure / Least operational overhead / Best performance / Highly available / Fault-tolerant
- **Constraints:** Budget, latency, compliance, data size, region, vpc, v.v.

## 4. ĐÁP ÁN ĐÚNG
**✅ Đáp án: [A/B/C/D]**

**Giải thích:**
- Tại sao đáp án này đúng
- Service nào được sử dụng và vai trò của nó
- Cách nó giải quyết yêu cầu của đề

## 5. CÁC ĐÁP ÁN SAI
**❌ Đáp án [A/B/C/D]:**
- Tại sao sai
- Khi nào thì đáp án này ĐÚNG (nếu có) → giúp phân biệt use case

**❌ Đáp án [A/B/C/D]:**
- Tương tự...

## 6. MẸO GHI NHỚ (Memory Hook)
🧠 *Một câu ngắn gọn, dễ nhớ hoặc association để nhớ nhanh đáp án đúng*

---

## Ví dụ mẫu Memory Hook
> 🧠 *"Multi-AZ = High Availability, Read Replica = Scalability"*
>
> 🧠 *"S3 Standard-IA = infrequent access, Glacier = archive"*
>
> 🧠 *"NAT Gateway = outbound only, IGW = inbound + outbound"*
