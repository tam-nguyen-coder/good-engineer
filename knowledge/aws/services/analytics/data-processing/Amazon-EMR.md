# Amazon EMR (Amazon Elastic MapReduce)

- Big Data Processing

- Optimized for processing, moving, and transforming large amounts of data.

# Types

## 1. EMR on EC2

## 2. EMR Serverless

- Fully managed serverless big data platform that simplifies running analytics frameworks such as Apache Spark and Apache Hive.

## 3. EMR on EKS

# Common Use Cases

- Interactive big data analytics.
- Real-time big data analytics.
- **ETL (Extract, Transform, Load)**.

# Keywords
- **Apache Hadoop / Apache Spark / Apache Hive** (Open-source big data frameworks)
- **Big Data Processing** (Petabyte-scale distributed data processing)
- **EMRFS (EMR File System)** (Accesses data directly from S3, separation of compute and storage)
- **Spot Instances** (Commonly used on Task nodes to minimize compute cost for transient workloads)
- **Primary / Core / Task Nodes** (Primary controls cluster; Core runs tasks & stores HDFS data; Task only runs tasks)
- **EMR Serverless** (Run Spark/Hive applications without managing cluster infrastructure)
- **EMR on EKS** (Run big data frameworks on managed Kubernetes clusters)
